# 1.4 开发环境配置：用 uv 管理项目

## 学习目标

- 理解 uv 是什么，为什么用它
- 学会用现代 uv 命令管理项目依赖
- 掌握项目文件结构和工作流

## 核心概念

### uv 是什么？

uv 是 2024 年发布的 Python 包和项目管理器，用 Rust 编写，比 pip **快 10-100 倍**。

**uv 的目标：一个工具替代所有现有工具。**

| 旧工具 | uv 替代 |
|--------|---------|
| pip | ✅ |
| pip-tools | ✅ |
| pipx | ✅ |
| poetry | ✅ |
| pyenv | ✅ |
| virtualenv | ✅ |
| twine | ✅ |

### 项目 vs 脚本 vs 工具

uv 支持三种工作模式：

| 模式 | 用途 | 命令 |
|------|------|------|
| **项目** | 多文件代码，有依赖关系 | `uv sync`, `uv add`, `uv run` |
| **脚本** | 单文件，带内联依赖声明 | `uv run script.py` |
| **工具** | 命令行工具，类似 pipx | `uvx cowsay "hello"` |

我们用的是 **项目模式**。

### 虚拟环境

虚拟环境就像给项目 **准备了一个独立的工具箱**。

**类比：** 每个房间有自己的工具箱。厨房的工具不放卧室，不会搞混。

## 原理解析

### 项目文件结构

用 `uv init` 创建项目后，会有这些文件：

```
项目目录/
├── pyproject.toml    ← 项目配置 + 依赖声明
├── uv.lock           ← 锁定文件（记录所有依赖的精确版本）
├── .python-version   ← Python 版本
└── .venv/            ← 虚拟环境（自动生成）
```

| 文件 | 作用 | 要提交到 Git 吗？ |
|------|------|------------------|
| `pyproject.toml` | 声明依赖范围（比如 `numpy>=2.0`） | ✅ 是 |
| `uv.lock` | 锁定精确版本（比如 `numpy==2.4.4`） | ✅ 是 |
| `.python-version` | 指定 Python 版本 | ✅ 是 |
| `.venv/` | 虚拟环境（本地安装的文件） | ❌ 否 |

### 依赖的三种类型

```toml
[project]
dependencies = [           # ← 发布时需要的依赖（必装）
    "numpy>=2.0",
    "matplotlib>=3.9",
]

[dependency-groups]
dev = [                    # ← 开发时才需要的依赖
    "ruff>=0.9",
    "pytest>=8.0",
]

[project.optional-dependencies]
plot = [                   # ← 可选依赖（extras）
    "seaborn>=0.13",
]
```

| 类型 | 位置 | 何时安装 | 发布时包含？ |
|------|------|---------|------------|
| 必装依赖 | `[project].dependencies` | 总是 | ✅ 是 |
| 开发依赖 | `[dependency-groups].dev` | 用 `--all-groups` | ❌ 否 |
| 可选依赖 | `[project.optional-dependencies]` | 用 `--extra` | ⚠️ 用户可选 |

### 核心命令

| 命令 | 作用 |
|------|------|
| `uv sync` | 同步依赖（解析依赖 → 更新 lock → 安装到 .venv） |
| `uv sync --all-groups` | 同步依赖 + 开发依赖 |
| `uv add <package>` | 添加新依赖（自动更新 pyproject.toml + uv.lock） |
| `uv remove <package>` | 移除依赖 |
| `uv run <command>` | 在虚拟环境中运行命令（自动 lock + sync） |
| `uv lock` | 只更新锁定文件，不安装 |
| `uv tree` | 查看依赖树 |

### `uv sync` vs `uv pip install`

| `uv sync`（现代方式） | `uv pip install`（旧兼容模式） |
|----------------------|---------------------------|
| 读取 `pyproject.toml` | 读取 `requirements.txt` |
| 自动管理 `uv.lock` | 需要手动 `uv pip compile` |
| 自动创建/管理 `.venv` | 需要手动 `uv venv` |
| 支持开发依赖分组 | 不支持 |
| **推荐用于项目** | 推荐用于迁移旧项目 |

### `uv run` 的自动机制

`uv run` 是"一键运行"命令。它会自动：

1. 检查 `pyproject.toml` 是否比 `uv.lock` 新
2. 如果是 → 自动重新锁定依赖（`uv lock`）
3. 检查 `.venv` 是否缺少包
4. 如果是 → 自动同步环境（`uv sync`）
5. 运行你的命令

**好处：** 你只需要写 `uv run python xxx.py`，不需要手动 lock 和 sync。

## 代码实战

### 完整的环境搭建

```bash
# 第 1 步：进入项目目录
cd hello-deep-learning

# 第 2 步：同步所有依赖（自动创建 .venv）
uv sync --all-groups
```

输出示例：
```
Using CPython 3.13.0
Creating virtual environment at: .venv
Resolved 45 packages in 12ms
Installed 42 packages in 1.8s
```

### 验证安装

```bash
# 查看安装了什么
uv pip list

# 测试导入
uv run python -c "import numpy; print('NumPy:', numpy.__version__)"
```

### 运行代码

```bash
# 推荐：用 uv run（自动确保依赖正确）
uv run python ch01-python基础/code/hungry.py

# 传统方式：先激活环境
source .venv/bin/activate
python ch01-python基础/code/hungry.py
```

### 管理依赖

```bash
# 添加新依赖
uv add pandas           # 加到必装依赖
uv add --dev pytest     # 加到开发依赖
uv add --extra plot seaborn  # 加到可选依赖

# 移除依赖
uv remove pandas

# 升级单个包
uv lock --upgrade-package numpy

# 查看所有依赖
uv tree
```

### 启动 Jupyter

```bash
uv run jupyter notebook
# 或
uv run jupyter lab
```

## 避坑指南

1. **不要用 `uv pip install`** — 这是 pip 兼容模式，绕过 uv 的项目管理功能。新项目用 `uv sync` 和 `uv add`。

2. **`uv.lock` 必须提交到 Git** — 这个文件记录了所有依赖的精确版本，保证团队协作时环境一致。没有它，不同人可能装到不同版本。

3. **克隆项目后的第一步** — 只需运行 `uv sync --all-groups`，不需要手动装任何包。

4. **`uv run` 默认不装开发依赖** — 如果你需要用 pytest 等开发工具，用 `uv run --all-groups pytest` 或先 `uv sync --all-groups`。

5. **Windows 激活路径不同** — Windows 用 `.venv\Scripts\activate`。
