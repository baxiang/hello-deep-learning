# 1.4 开发环境配置：用 uv 管理项目

## 学习目标

- 理解什么是依赖和虚拟环境
- 学会用现代 uv 命令管理项目依赖
- 掌握 `uv sync` 和 `uv add` 的区别

## 核心概念

### 什么是"依赖"？

写代码就像做菜，需要各种"调料"（外部工具包）。

这个项目需要：
- **NumPy** → 做数学运算的"计算器"
- **Matplotlib** → 画图的"画笔"
- **Jupyter** → 交互式编程的"笔记本"

这些外部工具包就叫做 **依赖**。

### 什么是"虚拟环境"？

虚拟环境就像给项目 **准备了一个独立的工具箱**。

**类比：** 每个房间有自己的工具箱。厨房的工具不放卧室，不会搞混。

## 原理解析

### uv 是什么？

uv 是 2024-2026 年最流行的 Python 包管理器，比传统的 pip **快 10-100 倍**。

你可以把 uv 理解成：
- 前端：npm（JavaScript）
- 后端：Cargo（Rust）
- Python 版：uv

### 现代 uv 的核心命令

```
uv sync      → 同步项目依赖（相当于 npm install）
uv add       → 添加新依赖（相当于 npm install xxx）
uv remove    → 移除依赖（相当于 npm uninstall xxx）
uv run       → 在虚拟环境中运行命令
```

### 旧命令 vs 新命令

| 旧方式（2024 之前） | 现代方式（2025+） | 说明 |
|-------------------|------------------|------|
| `uv pip install -e ".[dev]"` | `uv sync --all-groups` | 安装所有依赖 |
| `uv pip install pandas` | `uv add pandas` | 添加新依赖 |
| `uv pip install -e ".[dev]"` | `uv sync` | 安装依赖+项目本身 |
| `python script.py` | `uv run python script.py` | 运行代码 |

**关键区别：**
- 旧方式：`uv pip` 是兼容模式，绕过 uv 的项目管理功能
- 新方式：`uv sync` / `uv add` 是原生命令，自动管理虚拟环境和依赖锁文件

### `pyproject.toml` 里的依赖定义

项目根目录的 `pyproject.toml` 定义了依赖关系：

```toml
[project]
dependencies = [
    "numpy>=2.0",        # 必装：数学运算
    "matplotlib>=3.9",   # 必装：画图
    "jupyter>=1.1",      # 必装：交互式笔记本
]

[dependency-groups]
dev = [                  # 开发用（可选）
    "ruff>=0.9",         # 代码格式化
    "ipython>=9.0",      # 增强版交互环境
]
```

**`uv sync`** 会安装 `[project]` 里的必装依赖。

**`uv sync --all-groups`** 会额外安装 `[dependency-groups]` 里的开发依赖。

## 代码实战

### 完整的环境搭建流程

```bash
# 第 1 步：进入项目目录
cd hello-deep-learning

# 第 2 步：初始化项目（创建虚拟环境 + 安装所有依赖）
uv sync --all-groups
```

这条命令会自动：
1. 创建 `.venv/` 虚拟环境
2. 读取 `pyproject.toml` 里的依赖
3. 下载并安装所有包
4. 生成 `uv.lock` 锁文件（记录精确版本）

你会看到类似输出：
```
Using CPython 3.13.0
Creating virtual environment at: .venv
Resolved 45 packages in 12ms
Downloaded 38 packages in 1.8s
Installed 42 packages in 0.3s
```

### 验证安装

```bash
# 查看安装了什么
uv pip list

# 测试导入
uv run python -c "import numpy; print('NumPy 版本:', numpy.__version__)"
```

### 运行代码

```bash
# 方式 1：直接用 uv run（推荐）
uv run python ch01-python基础/code/hungry.py

# 方式 2：先激活环境
source .venv/bin/activate
python ch01-python基础/code/hungry.py
```

### 以后安装新包

```bash
# 添加新依赖
uv add pandas

# 添加开发依赖
uv add --dev pytest

# 移除依赖
uv remove pandas
```

运行 `uv add` 后，`pyproject.toml` 和 `uv.lock` 会自动更新。

### 启动 Jupyter

```bash
uv run jupyter notebook
# 或
uv run jupyter lab
```

## 避坑指南

1. **不要用 `uv pip install`** — 这是旧兼容模式。新项目应该用 `uv sync` 和 `uv add`。

2. **`uv.lock` 文件要提交** — 这个文件记录了所有依赖的精确版本，保证团队协作时环境一致。

3. **克隆项目后的第一步** — 别人克隆项目后，只需运行 `uv sync --all-groups`，不需要手动装任何包。

4. **Windows 激活路径不同** — Windows 下用 `.venv\Scripts\activate` 而不是 `source .venv/bin/activate`。
