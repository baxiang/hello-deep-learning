# 1.4 开发环境配置：理解 `uv pip install -e ".[dev]"`

## 学习目标

- 理解这条命令的每个部分是什么意思
- 学会如何安装项目依赖
- 掌握虚拟环境的基本概念

## 核心概念

### 什么是"依赖"？

写代码就像做菜，需要各种"调料"（外部工具包）。

比如这个项目需要：
- **NumPy** → 做数学运算的"计算器"
- **Matplotlib** → 画图的"画笔"
- **Jupyter** → 交互式编程的"笔记本"

这些外部工具包就叫做 **依赖**。

### 什么是"虚拟环境"？

虚拟环境就像给你的项目 **准备了一个独立的工具箱**。

**类比：** 你装修房子，每个房间有自己的工具箱。厨房的工具不放卧室，卧室的工具不放客厅。这样不会搞混，也不会互相影响。

**为什么需要虚拟环境？**
- 项目 A 需要 NumPy 1.21，项目 B 需要 NumPy 1.25
- 没有虚拟环境 → 装来装去会冲突
- 有了虚拟环境 → 每个项目各用各的，互不影响

## 原理解析

### 拆解命令：`uv pip install -e ".[dev]"`

这条命令看起来很长，其实可以拆成 5 个部分：

```
uv    pip    install    -e    ".[dev]"
 │      │        │        │       │
 └──┬───┘└──┬────┘└──┬────┘└──┬──┘└──┬──┘
    │       │        │       │      │
  工具    子命令    动作    参数   目标
```

#### 1. `uv` — 快速包管理器

`uv` 是一个 **下载和安装 Python 工具包的程序**。

**类比：** 就像你手机上的"应用商店"，想装什么软件就去商店下载。

为什么用 `uv` 而不用传统的 `pip`？因为 **uv 快 10-100 倍**。装同样的东西，pip 要 1 分钟，uv 只要几秒钟。

#### 2. `pip` — 子命令

`pip` 是 Python 自带的包安装工具。

`uv pip` 的意思是："用 uv 的速度，但用 pip 的用法"。

#### 3. `install` — 安装

告诉程序："我要安装东西"。

#### 4. `-e` — 可编辑模式

`-e` 是 `--editable` 的缩写，意思是 **"可编辑模式"**。

这是什么意思呢？

**普通安装：** 把你的代码复制一份到 Python 的系统目录。你改了代码，系统目录里的不会变。

**可编辑模式安装：** 不复制代码，而是在系统目录里放一个"快捷方式"。你改了代码，立刻生效。

**类比：**
- 普通安装 → 把照片洗出来放到相册，改原图相册里的不变
- 可编辑模式 → 相册里放的是原图链接，改原图相册里的也变

**为什么要用 `-e`？** 因为项目本身定义了依赖关系，用 `-e` 安装后，以后加了新依赖，不用重新安装。

#### 5. `".[dev]"` — 安装目标

这个部分最复杂，再拆成 3 个小部分：

```
"  .  [dev]  "
│  │   │     │
└┬─┘└┬─┘└─┬──┘
 │    │    │
引号  当前  可选包
      目录  组
```

- **`.`** — 当前目录（就是你运行命令时所在的文件夹）
- **`[dev]`** — 安装"开发用"的额外依赖
- **引号** — 防止 Shell 误解方括号

### `pyproject.toml` 里的定义

在项目根目录有一个 `pyproject.toml` 文件，里面写着依赖关系：

```toml
[project]
dependencies = [
    "numpy>=1.21.0",         # 必装：数学运算
    "matplotlib>=3.4.0",     # 必装：画图
    "jupyter>=1.0.0",        # 必装：交互式笔记本
    "ipykernel>=6.0.0",      # 必装：Jupyter 内核
]

[project.optional-dependencies]
dev = [                      # dev 组（可选）
    "ruff>=0.1.0",           # 代码格式化
    "ipython>=8.0.0",        # 增强版 Python 交互环境
]
```

所以 `.[dev]` 的意思是：
1. 安装当前项目定义的所有 **必装依赖**（numpy、matplotlib、jupyter、ipykernel）
2. **再额外安装** `dev` 组里的依赖（ruff、ipython）

如果只写 `.` 不写 `[dev]`，就只装必装依赖，不装开发工具。

## 代码实战

### 完整的环境搭建流程

```bash
# 第 1 步：进入项目目录
cd hello-deep-learning

# 第 2 步：创建虚拟环境（第一次需要）
uv venv
```

这会在项目里创建一个 `.venv/` 文件夹，里面就是独立的 Python 环境。

```bash
# 第 3 步：激活虚拟环境
source .venv/bin/activate
```

激活后，你的命令行提示符前面会多一个 `(.venv)`，说明你现在用的是这个项目的独立环境。

```bash
# 第 4 步：安装所有依赖
uv pip install -e ".[dev]"
```

你会看到类似这样的输出：
```
Resolved 45 packages in 32ms
Downloaded 38 packages in 2.1s
Installed 42 packages in 0.5s
```

**解读：** uv 在 32 毫秒内找到了所有需要安装的包，2.1 秒下载完成，0.5 秒安装完成。如果用 pip，可能要等几十秒。

### 验证安装是否成功

```bash
# 查看安装了哪些包
pip list
```

应该能看到：
```
numpy           2.x.x
matplotlib      3.x.x
jupyter         1.x.x
ruff            0.x.x
...
```

```bash
# 测试 NumPy 能否导入
python -c "import numpy; print('NumPy 版本:', numpy.__version__)"
```

```bash
# 测试 Jupyter 能否启动
jupyter --version
```

### 以后安装新包

如果后面发现需要新的工具包，比如 `pandas`：

```bash
# 方法 1：用 uv 直接安装（临时用）
uv pip install pandas

# 方法 2：写入配置文件（推荐，别人克隆项目后能自动安装）
# 编辑 pyproject.toml，在 dependencies 里加 "pandas>=2.0.0"
# 然后重新运行
uv pip install -e ".[dev]"
```

## 避坑指南

1. **忘记激活虚拟环境** — 运行 `source .venv/bin/activate` 前，所有安装都会装到系统 Python 里，而不是项目环境。检查方法：看命令行提示符前面有没有 `(.venv)`。

2. **命令中的引号不能省** — `.[dev]` 外面的引号必须写。如果不写引号，Shell 会把 `[dev]` 当成特殊字符处理，报错 `bash: .[dev]: event not found`。

3. **Windows 系统激活命令不同** — Windows 下激活虚拟环境用 `.venv\Scripts\activate`，而不是 `source .venv/bin/activate`。

4. **不要手动改 `.venv/` 里的文件** — 虚拟环境里的文件都是自动管理的，手动改可能导致环境损坏。
