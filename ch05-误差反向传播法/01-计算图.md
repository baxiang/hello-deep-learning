# 5.1 计算图

## 学习目标

- 理解计算图的基本概念
- 学会用计算图表示复杂的数学运算
- 掌握计算图的正向传播和反向传播

## 核心概念

### 什么是计算图？

计算图是一种 **把计算过程可视化的工具**。

**生活中的类比：** 假设你要买 3 个苹果，每个苹果 100 元，还要交 10% 的消费税。

你会怎么算？
1. 苹果总价 = 100 × 3 = 300 元
2. 含税价格 = 300 × 1.1 = 330 元

计算图就是把每一步计算画成节点，用箭头连起来：

```
苹果单价(100) ──→ [×] ──→ 300 ──→ [×] ──→ 330(含税)
                ↑                    ↑
苹果数量(3)  ───┘     税率(1.1) ────┘
```

### 计算图的好处

1. **局部集中** - 每个节点只关心自己的输入和输出，不需要知道全局。
2. **可复用** - 同样的计算（比如乘法）可以用在很多不同的场景。
3. **反向传播** - 计算图不仅能从前往后算，还能从后往前"倒推"。

## 原理解析

### 正向传播（Forward Propagation）

正向传播就是 **从左到右** 计算。输入数据从左边进入，经过每个节点的计算，从右边输出结果。

上面的例子中：
- 第一个乘法节点：接收 100 和 3，输出 300。
- 第二个乘法节点：接收 300 和 1.1，输出 330。

### 反向传播（Backward Propagation）

反向传播就是 **从右到左** 倒推。假设你想知道"如果苹果单价涨了 1 元，最终价格会涨多少"。

从右边的输出 330 开始倒推：
- 第二个乘法节点的导数是 1.1（因为 d(300×1.1)/d(300) = 1.1）。
- 所以 300 变化 1，最终结果变化 1.1。
- 第一个乘法节点的导数是 3（因为 d(100×3)/d(100) = 3）。
- 所以 100 变化 1，300 变化 3，最终结果变化 3 × 1.1 = 3.3。

### 计算图的核心规则

每个节点有两个功能：
1. **forward（正向）** - 接收输入，计算输出。
2. **backward（反向）** - 接收上游传来的导数，乘以本地导数，传给下游。

## 代码实战

```python
class MulLayer:
    """乘法层"""

    def __init__(self):
        self.x = None
        self.y = None

    def forward(self, x, y):
        self.x = x
        self.y = y
        return x * y

    def backward(self, dout):
        dx = dout * self.y    # 翻转 x 和 y
        dy = dout * self.x
        return dx, dy

class AddLayer:
    """加法层"""

    def __init__(self):
        pass

    def forward(self, x, y):
        return x + y

    def backward(self, dout):
        dx = dout * 1
        dy = dout * 1
        return dx, dy

# 示例：买 2 个苹果和 3 个橘子
apple = 100
apple_num = 2
orange = 150
orange_num = 3
tax = 1.1

# 创建层
mul_apple_layer = MulLayer()
mul_orange_layer = MulLayer()
add_fruit_layer = AddLayer()
mul_tax_layer = MulLayer()

# 正向传播
apple_price = mul_apple_layer.forward(apple, apple_num)
orange_price = mul_orange_layer.forward(orange, orange_num)
total_price = add_fruit_layer.forward(apple_price, orange_price)
final_price = mul_tax_layer.forward(total_price, tax)

print("最终价格:", final_price)  # (100*2 + 150*3) * 1.1 = 715

# 反向传播：求各个变量对最终价格的影响
dprice = 1.0  # 假设最终价格的导数为 1
dtotal_price, dtax = mul_tax_layer.backward(dprice)
dapple_price, dorange_price = add_fruit_layer.backward(dtotal_price)
dapple, dapple_num = mul_apple_layer.backward(dapple_price)
dorange, dorange_num = mul_orange_layer.backward(dorange_price)

print("\n=== 反向传播结果 ===")
print("dapple =", dapple)             # 苹果单价的导数: 2 * 1.1 = 2.2
print("dapple_num =", dapple_num)     # 苹果数量的导数: 100 * 1.1 = 110
print("dorange =", dorange)           # 橘子单价的导数: 3 * 1.1 = 3.3
print("dorange_num =", dorange_num)   # 橘子数量的导数: 150 * 1.1 = 165
print("dtax =", dtax)                 # 税率的导数: 100*2 + 150*3 = 650
```

**代码解读：**
- `MulLayer` 是乘法层，`forward` 做乘法并保存输入，`backward` 计算导数（乘法法则：d(xy)/dx = y，d(xy)/dy = x）。
- `AddLayer` 是加法层，加法的导数是 1，所以 `backward` 直接把上游导数原样传下去。
- 正向传播：先算苹果总价、橘子总价，再相加，最后乘以税率。
- 反向传播：从最终价格开始，依次往回推算每个变量的导数。`dout` 是从上游传来的导数。
- `dapple = 2.2` 表示苹果单价每涨 1 元，最终价格涨 2.2 元（因为买了 2 个，还要交 10% 的税）。

## 避坑指南

1. **忘记保存正向传播的输入** - `backward` 需要用到正向传播时的输入值（比如 `self.x`, `self.y`）。如果没保存，反向传播就算不了。
2. **导数方向搞反** - 反向传播是从右往左，`dout` 是从上游（右边）传来的导数。
3. **乘法层的导数要"翻转"** - `dx = dout * self.y`，`dy = dout * self.x`，注意是乘对方的值，不是自己的值。
