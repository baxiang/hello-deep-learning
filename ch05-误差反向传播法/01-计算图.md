# 5.1 计算图

## 学习目标

- 理解计算图是什么
- 学会用计算图表示日常计算
- 掌握正向传播和反向传播的概念

## 核心概念

### 什么是计算图？

计算图就是 **把计算过程画成一张图**。

每个计算步骤是一个"节点"，用箭头把它们连起来。

**生活中的例子：买苹果**

你要买 3 个苹果，每个 100 元，还要交 10% 的消费税。

你怎么算的？
1. 苹果总价 = 100 × 3 = 300 元
2. 含税价格 = 300 × 1.1 = 330 元

计算图长这样：

```
苹果单价(100) ──→ [乘法] ──→ 300 ──→ [乘法] ──→ 330(含税)
                 ↑                      ↑
苹果数量(3)  ────┘       税率(1.1) ─────┘
```

**这到底意味着什么？**

计算图就是把"先算什么、后算什么"画出来。每个方框是一个计算步骤，箭头是数据的流向。

### 计算图的两个方向

**正向传播（从左到右）：** 算结果。

```
100 × 3 = 300 → 300 × 1.1 = 330
```

**反向传播（从右到左）：** 算"如果某个输入变了，结果会变多少"。

```
330 ← 300 的变化影响 ← 100 的变化影响
```

**类比：**
- 正向传播 = 做菜：食材 → 加工 → 成品
- 反向传播 = 试菜后改进：成品不好吃 → 哪个步骤有问题 → 哪种食材要调整

## 原理解析

### 乘法节点的反向传播

这是计算图最关键的部分。

假设你想知道：**如果苹果单价涨了 1 元，最终价格会涨多少？**

从右往左推：
1. 含税价格对总价的导数 = 1.1（税率）
2. 总价对单价的导数 = 3（数量）
3. 所以单价涨 1 元 → 最终价格涨 3 × 1.1 = 3.3 元

**这到底意味着什么？**

买了 3 个苹果，单价涨 1 元，总价涨 3 元。再乘以税率 1.1，最终涨 3.3 元。

**乘法节点的规则：** 反向传播时，把输入的数值"翻转"传给对方。

```
x × y = z

正向：x 和 y 进去，z 出来
反向：z 的导数进来 → x 那边收到 z的导数 × y，y 那边收到 z的导数 × x
```

### 加法节点的反向传播

加法更简单：**导数原样通过**。

```
x + y = z

正向：x 和 y 进去，z 出来
反向：z 的导数进来 → x 和 y 各收到同样的导数
```

**为什么？** 因为 x 涨 1，z 也涨 1；y 涨 1，z 也涨 1。每个输入对输出的影响是 1:1 的。

## 代码实战

### 完整可运行代码

```python
import numpy as np

# === 1. 乘法层 ===
class MulLayer:
    """乘法节点"""
    def __init__(self):
        self.x = None
        self.y = None

    def forward(self, x, y):
        """正向传播：计算 x × y"""
        self.x = x
        self.y = y
        return x * y

    def backward(self, dout):
        """反向传播：dout 是从上游传来的导数"""
        dx = dout * self.y   # x 的导数 = 上游导数 × y
        dy = dout * self.x   # y 的导数 = 上游导数 × x
        return dx, dy

# === 2. 加法层 ===
class AddLayer:
    """加法节点"""
    def __init__(self):
        pass

    def forward(self, x, y):
        """正向传播：计算 x + y"""
        return x + y

    def backward(self, dout):
        """反向传播：导数原样通过"""
        dx = dout * 1
        dy = dout * 1
        return dx, dy

# === 3. 示例：买 2 个苹果和 3 个橘子，加 10% 税 ===
apple = 100        # 苹果单价
apple_num = 2      # 苹果数量
orange = 150       # 橘子单价
orange_num = 3     # 橘子数量
tax = 1.1          # 税率

# 创建计算节点
mul_apple = MulLayer()
mul_orange = MulLayer()
add_fruit = AddLayer()
mul_tax = MulLayer()

# 正向传播：从左到右算结果
apple_price = mul_apple.forward(apple, apple_num)     # 100 × 2 = 200
orange_price = mul_orange.forward(orange, orange_num) # 150 × 3 = 450
total_price = add_fruit.forward(apple_price, orange_price)  # 200 + 450 = 650
final_price = mul_tax.forward(total_price, tax)       # 650 × 1.1 = 715

print(f"苹果总价: {apple_price}")
print(f"橘子总价: {orange_price}")
print(f"水果合计: {total_price}")
print(f"含税价格: {final_price}")

# 反向传播：从右到左算导数
dprice = 1.0  # 假设最终价格变化 1
dtotal_price, dtax = mul_tax.backward(dprice)
dapple_price, dorange_price = add_fruit.backward(dtotal_price)
dapple, dapple_num = mul_apple.backward(dapple_price)
dorange, dorange_num = mul_orange.backward(dorange_price)

print(f"\n=== 反向传播结果 ===")
print(f"苹果单价的导数: {dapple} (单价涨 1 元，最终涨 {dapple} 元)")
print(f"苹果数量的导数: {dapple_num}")
print(f"橘子单价的导数: {dorange}")
print(f"橘子数量的导数: {dorange_num}")
print(f"税率的导数: {dtax} (税率涨 1，最终涨 {dtax} 元)")
```

### 运行结果

```
苹果总价: 200
橘子总价: 450
水果合计: 650
含税价格: 715.0

=== 反向传播结果 ===
苹果单价的导数: 2.2 (单价涨 1 元，最终涨 2.2 元)
苹果数量的导数: 110.0
橘子单价的导数: 3.3
橘子数量的导数: 165.0
税率的导数: 650.0 (税率涨 1，最终涨 650.0 元)
```

**解读：**
- 苹果单价涨 1 元 → 最终涨 2.2 元（买了 2 个 × 税率 1.1）
- 橘子数量涨 1 个 → 最终涨 165 元（单价 150 × 税率 1.1，再加原来的橘子）
- 税率涨 1 → 最终涨 650 元（水果合计 × 1）

### 逐行解读

**乘法层正向：**
```python
def forward(self, x, y):
    self.x = x
    self.y = y
    return x * y
```
1. 保存 x 和 y（反向传播时要用）。
2. 返回 x × y。

**乘法层反向：**
```python
def backward(self, dout):
    dx = dout * self.y
    dy = dout * self.x
    return dx, dy
```
1. `dout` 是从上游（右边）传来的导数。
2. x 的导数 = dout × y（翻转）。
3. y 的导数 = dout × x（翻转）。

## 避坑指南

1. **正向传播时必须保存输入** - `backward` 需要用到 `self.x` 和 `self.y`。如果没保存，反向传播就算不了。

2. **反向传播从右往左** - 先调用最右边的节点的 `backward`，然后把结果传给左边的节点。

3. **乘法的导数要"翻转"** - x 那边的导数乘的是 y，不是 x。这是最容易搞错的地方。
