# 9.1 序列数据是什么

## 学习目标

- 理解什么是序列数据
- 认识常见的序列数据类型
- 了解序列数据与普通数据的区别

## 核心概念

### 什么是序列数据？

**序列数据就是"有顺序"的数据。**

想象你在看电影：
- 单张图片 → 普通数据（无所谓先后）
- 视频片段 → 序列数据（帧的顺序很重要）

序列数据的关键特点：**顺序不能乱，顺序变了意思就变了。**

比如句子"我喜欢你"和"你喜欢我"，字都一样，顺序不同，意思完全不同。

### 常见的序列数据

| 类型 | 例子 | 特点 |
|------|------|------|
| 文本 | "今天天气真好" | 字符/词语有顺序 |
| 语音 | 说话录音 | 声波随时间变化 |
| 视频 | 电影片段 | 图像帧序列 |
| 股票 | 每日股价 | 时间序列数据 |
| 天气 | 每日温度 | 时间序列数据 |

### 为什么普通神经网络处理不了？

普通神经网络（全连接网络、CNN）假设：
- 输入之间相互独立
- 输入顺序不影响结果

这对于图片分类没问题（把图片打乱像素还能认出来），但对序列数据就不行了。

**普通网络没法"记住"之前看过的内容。**

比如翻译句子，翻译"我喜欢"时，网络得记住前面看过"我"和"喜欢"，才能准确翻译成 "I like"。

## 原理解析

### 序列数据的特点

1. **顺序性**：元素之间的相对位置重要
2. **变长性**：不同样本长度可能不同
3. **依赖性**：当前元素可能与前面/后面的元素相关

### 数学表示

一个序列可以表示为：
```
X = (x₁, x₂, x₃, ..., xₜ, ..., xₜ)
```

其中：
- xₜ 是第 t 个时间步的输入
- T 是序列总长度

### 序列建模的目标

给定序列，我们可能想要：
- **预测下一个**：输入"今天天气"，预测"真"
- **分类整个序列**：判断这句话是褒义还是贬义
- **序列到序列**：翻译、摘要生成

### 为什么需要专门的模型？

| 模型类型 | 能处理序列吗？ | 原因 |
|----------|---------------|------|
| 全连接网络 | ❌ | 输入长度固定，无记忆 |
| CNN | 部分 | 可以处理局部模式，但难以建模长距离依赖 |
| RNN | ✅ | 有记忆，能处理变长序列 |
| Transformer | ✅ | 通过注意力机制建模全局依赖 |

## 代码实战

### 序列数据表示

```python
import numpy as np

# === 1. 文本序列 ===
# 把句子转换成数字序列（简单示例）
sentence = "我喜欢深度学习"
char_to_idx = {char: i for i, char in enumerate(set(sentence))}
print("字符到索引:", char_to_idx)

# 序列编码
sequence = [char_to_idx[c] for c in sentence]
print("编码后的序列:", sequence)

# === 2. 时间序列 ===
# 模拟股票价格（100天的数据）
np.random.seed(42)
days = 100
trend = np.linspace(100, 150, days)  # 上涨趋势
noise = np.random.randn(days) * 5    # 随机波动
prices = trend + noise
print(f"\n股票价格序列: {prices[:5]}... (共{len(prices)}天)")

# === 3. 滑动窗口构建训练数据 ===
def create_sequences(data, window_size):
    """把时间序列变成 (输入, 输出) 对"""
    X, y = [], []
    for i in range(len(data) - window_size):
        X.append(data[i:i+window_size])
        y.append(data[i+window_size])
    return np.array(X), np.array(y)

window_size = 7
X, y = create_sequences(prices, window_size)
print(f"\n滑动窗口示例 (窗口大小={window_size}):")
print(f"输入形状: {X.shape}  # (样本数, 窗口大小)")
print(f"输出形状: {y.shape}  # 预测下一个值")
print(f"第一个样本: 输入={X[0]}, 输出={y[0]:.2f}")
```

### 运行结果

```

编码后的序列: [0, 1, 2, 3, 4, 5, 6]

股票价格序列: [102.49671415 101.8617357  108.64768716 ... (共100天)

滑动窗口示例 (窗口大小=7):
输入形状: (93, 7)  # (样本数, 窗口大小)
输出形状: (93,)  # 预测下一个值
第一个样本: 输入=[102.5  101.86 108.65 109.16 109.84 110.38 110.67], 输出=109.96
```

## 避坑指南

1. **混淆序列顺序**：处理序列数据时，永远不要打乱时间顺序！数据增强时要注意这一点。

2. **忽略变长问题**：不同样本长度不同时，需要填充（padding）或截断，但要区分有效数据和填充数据。

3. **只看当前不看历史**：用普通网络处理序列时，容易只把当前输入当作特征，忽略了历史信息。
