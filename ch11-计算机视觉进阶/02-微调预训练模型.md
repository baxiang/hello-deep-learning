- 理解迁移学习的核心思想

- 了解预训练模型的选择

## 核心概念

### 迁移学习：站在巨人的肩膀上

**从头训练模型需要大量数据和计算资源。**

比如训练一个图像分类模型：
- 需要百万张图片
- 需要GPU 训练数周
- 普通人很难做到

**迁移学习的思路：借用别人训练好的模型。**

别人用百万图片训练的模型，已经学会了识别各种物体。我们可以借用它的能力，稍作调整，用于自己的任务。

**类比：学习新技能。**

学开车 → 不需要从力学原理开始学
→ 直接学习操控技巧（借用已有的力学知识）

### 微调：让预训练模型适应新任务

**预训练模型**：在大数据集上训练好的模型（如ImageNet）
**微调**：用小数据集稍作调整，让它适应新任务

微调流程：
```
1. 加载预训练模型（已学会识别1000类物体）
2. 替换最后一层（改成你的类别数）
3. 用你的数据继续训练（调整部分参数）
4. 得到适应新任务的模型
```

### 为什么微调有效？

预训练模型已经学到了**通用特征**：

| 层级 | 学到的特征 | 可迁移性 |
|------|-----------|----------|
| 底层 | 边缘、纹理 | 高（所有图像都有） |
|中层 | 形状、图案 | 中（大部分图像通用） |
| 高层 | 物体部件 | 低（特定类别相关） |
| 输出层 | 类别判定 | 需要替换 |

**底层特征是通用的，可以直接借用。**

## 原理解析

### 微调的策略

**1. 特征提取（冻结所有层）**

只替换最后一层，其他层参数不变。
- 优点：快速、稳定
- 缺点：可能不够灵活
- 适用：数据量很小、任务相似度高

**2. 微调部分层**

冻结底层，只训练高层。
- 优点：平衡效率和灵活性
- 适用：中等数据量

**3. 微调全部层**

所有层都参与训练（底层用小学习率）。
- 优点：最灵活
- 缺点：可能过拟合、需要更多数据
- 适用：大数据量、任务差异大

### 学习率的设置

微调时，不同层用不同学习率：

```
底层：小学习率（如 0.001） → 保持已学到的特征
高层：大学习率（如 0.01） → 快速适应新任务
新层：更大学习率（如 0.1） → 从头学习
```

### 常用预训练模型

| 模型 | 特点 | 适用场景 |
|------|------|----------|
| ResNet | 深层网络，性能好 | 通用图像分类 |
| VGG | 结构简单，易理解 | 教学、简单任务 |
| EfficientNet | 高效，参数少 | 资源受限场景 |
| MobileNet | 轻量，速度快 | 移动端应用 |

## 代码实战

### 微调流程示意（概念演示）

```python
import numpy as np

class PretrainedModel:
    """模拟预训练模型"""

    def __init__(self, num_original_classes=1000):
        # 模拟预训练权重（已学会识别1000类）
        np.random.seed(42)
        self.features = np.random.randn(512)  # 特征向量
        self.classifier_W = np.random.randn(512, num_original_classes) * 0.01
        self.classifier_b = np.zeros(num_original_classes)

    def extract_features(self, x):
        """提取特征（这部分可以冻结）"""
        # 模拟特征提取
        return np.random.randn(512)

    def predict(self, x):
        """原始预测"""
        features = self.extract_features(x)
        logits = np.dot(features, self.classifier_W) + self.classifier_b
        return logits


class FineTunedModel:
    """微调后的模型"""

    def __init__(self, pretrained, num_new_classes):
        """初始化微调模型

        Args:
            pretrained: 预训练模型
            num_new_classes: 新任务的类别数
        """
        self.pretrained = pretrained

        # 替换最后一层
        np.random.seed(42)
        self.new_classifier_W = np.random.randn(512, num_new_classes) * 0.1
        self.new_classifier_b = np.zeros(num_new_classes)

    def predict(self, x):
        """微调后的预测"""
        # 使用预训练模型的特征提取
        features = self.pretrained.extract_features(x)

        # 使用新的分类器
        logits = np.dot(features, self.new_classifier_W) + self.new_classifier_b
        return logits


# === 演示 ===
print("=== 微调预训练模型演示 ===")

# 加载预训练模型（原本识别1000类）
pretrained = PretrainedModel(num_original_classes=1000)
print(f"预训练模型: 原始类别数 = 1000")

# 微调为新任务（识别5类）
finetuned = FineTunedModel(pretrained, num_new_classes=5)
print(f"微调后模型: 新类别数 = 5")

# 模拟输入
x = np.random.randn(224, 224, 3)

# 对比输出
original_output = pretrained.predict(x)
new_output = finetuned.predict(x)

print(f"\n原始输出形状: {original_output.shape}")
print(f"微调后输出形状: {new_output.shape}")

print("\n=== 微调的关键步骤 ===")
print("1. 加载预训练模型的权重")
print("2. 冻结底层特征提取层（可选）")
print("3. 替换分类层为新类别数")
print("4. 用新数据训练，调整参数")
print("5. 底层用小学习率，高层用大学习率")
```

## 避坑指南

1. **忘记替换输出层**：预训练模型输出1000类，你的任务可能只有5类。

2. **学习率设置不当**：用太大学习率会破坏预训练的特征。

3. **数据预处理不一致**：预训练模型用特定的预处理方式，微调时要保持一致。

4. **过早停止训练**：微调需要足够的时间让高层适应新任务。
