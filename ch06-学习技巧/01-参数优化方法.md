# 6.1 参数优化方法

## 学习目标

- 理解什么是优化器
- 掌握 SGD、Momentum、AdaGrad、Adam 四种方法
- 能直观理解每种方法的特点

## 核心概念

### 什么是优化器？

优化器就是 **用梯度更新权重的策略**。

第 4 章学了一个最简单的：

```
新权重 = 旧权重 - 学习率 × 梯度
```

这就像蒙眼下山：感觉哪个方向最陡，就往那走一步。

**但这个方法有个大问题：如果山路很曲折，会走很多冤枉路。**

所以我们需要更好的"下山策略"。这就是优化器的作用。

## 原理解析

### 1. SGD（随机梯度下降）

```
权重 = 权重 - 学习率 × 梯度
```

**类比：** 蒙眼下山，感觉哪边最陡就往哪迈一步。

**缺点：** 如果地形是"之"字形山谷，SGD 会来回撞墙，走得慢。

### 2. Momentum（动量）

Momentum 给 SGD 加了"惯性"。

```
速度 = 0.9 × 旧速度 - 学习率 × 梯度
权重 = 权重 + 速度
```

**类比：** 下山时你跑起来了。上次往东走了，这次即使梯度说往东北，你还是会偏东一点，因为有惯性。

**好处：** 方向一致时越跑越快；方向频繁变化时互相抵消，减少震荡。

### 3. AdaGrad（自适应学习率）

AdaGrad 给每个参数配了"个人学习率"。

```
历史记录 += 梯度²
权重 = 权重 - 学习率 × 梯度 / (√历史记录 + 小常数)
```

**类比：** 每个学生有自己的学习进度。经常错的题（梯度大），少做几遍（学习率小）；很少错的题（梯度小），多做几遍（学习率大）。

**好处：** 适合"有的参数经常更新，有的很少更新"的场景。

### 4. Adam（自适应矩估计）

Adam = Momentum 的惯性 + AdaGrad 的自适应。

**类比：** 你开车下山，既有惯性（Momentum），又根据路况自动调档位（AdaGrad）。

**这是目前最常用的优化器。** 如果不知道选哪个，就选 Adam。

## 代码实战

### 完整可运行代码

```python
import numpy as np

# === 1. SGD ===
class SGD:
    def __init__(self, lr=0.01):
        self.lr = lr

    def update(self, params, grads):
        for key in params:
            params[key] -= self.lr * grads[key]

# === 2. Momentum ===
class Momentum:
    def __init__(self, lr=0.01, momentum=0.9):
        self.lr = lr
        self.momentum = momentum
        self.v = None

    def update(self, params, grads):
        if self.v is None:
            self.v = {}
            for key in params:
                self.v[key] = np.zeros_like(params[key])

        for key in params:
            self.v[key] = self.momentum * self.v[key] - self.lr * grads[key]
            params[key] += self.v[key]

# === 3. AdaGrad ===
class AdaGrad:
    def __init__(self, lr=0.01):
        self.lr = lr
        self.h = None

    def update(self, params, grads):
        if self.h is None:
            self.h = {}
            for key in params:
                self.h[key] = np.zeros_like(params[key])

        for key in params:
            self.h[key] += grads[key] ** 2
            params[key] -= self.lr * grads[key] / (np.sqrt(self.h[key]) + 1e-7)

# === 4. Adam ===
class Adam:
    def __init__(self, lr=0.001, beta1=0.9, beta2=0.999):
        self.lr = lr
        self.beta1 = beta1
        self.beta2 = beta2
        self.iter = 0
        self.m = None
        self.v = None

    def update(self, params, grads):
        if self.m is None:
            self.m, self.v = {}, {}
            for key in params:
                self.m[key] = np.zeros_like(params[key])
                self.v[key] = np.zeros_like(params[key])

        self.iter += 1
        lr_t = self.lr * np.sqrt(1 - self.beta2 ** self.iter) / \
               (1 - self.beta1 ** self.iter)

        for key in params:
            self.m[key] += (1 - self.beta1) * (grads[key] - self.m[key])
            self.v[key] += (1 - self.beta2) * (grads[key] ** 2 - self.v[key])
            params[key] -= lr_t * self.m[key] / (np.sqrt(self.v[key]) + 1e-7)

# === 测试 ===
def f(x):
    """目标函数: x0²/20 + x1²"""
    return x[0]**2 / 20 + x[1]**2

def grad_f(x):
    """目标函数的梯度"""
    return np.array([x[0] / 10, 2 * x[1]])

print("=== 优化器对比测试 ===")
print(f"目标: 找到 f(x) = x0²/20 + x1² 的最小值")
print(f"初始位置: [-7, 2], 初始值: {f(np.array([-7, 2])):.4f}")
print(f"最优位置: [0, 0], 最优值: 0\n")

for name, optimizer in [
    ("SGD", SGD(lr=0.95)),
    ("Momentum", Momentum(lr=0.1)),
    ("AdaGrad", AdaGrad(lr=1.5)),
    ("Adam", Adam(lr=0.3)),
]:
    x = np.array([-7.0, 2.0])
    for i in range(30):
        g = grad_f(x)
        optimizer.update({'x': x}, {'x': g})

    print(f"{name:10s}: 最终位置 [{x[0]:6.2f}, {x[1]:6.2f}], "
          f"函数值 {f(x):.6f}")
```

### 运行结果

```

初始位置: [-7, 2], 初始值: 6.4500
最优位置: [0, 0], 最优值: 0

SGD       : 最终位置 [ -0.31,  -0.01], 函数值 0.004829
Momentum  : 最终位置 [ -0.00,   0.00], 函数值 0.000000

Adam      : 最终位置 [ -0.00,   0.00], 函数值 0.000000
```

**解读：**
- SGD 走了 30 步还没到最优（0.0048），因为它会来回震荡。
- Momentum、AdaGrad、Adam 都到了最优附近（0.00000x）。
- Adam 综合性能最好，是默认选择。

## 避坑指南

1. **学习率最重要** - 不管用哪个优化器，学习率设错了都不行。通常先试 0.001（Adam）或 0.01（SGD）。

2. **SGD 不是不好，而是慢** - 给足时间，SGD 也能找到好结果。只是比其他方法慢。

3. **Adam 也不是万能的** - 有些研究发现 SGD 在某些情况下泛化能力更好（测试集表现更好）。
