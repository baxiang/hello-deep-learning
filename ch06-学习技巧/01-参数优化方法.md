# 6.1 参数优化方法

## 学习目标

- 理解为什么需要不同的优化方法
- 掌握 SGD、Momentum、AdaGrad、Adam 四种优化器
- 理解每种优化器的优缺点

## 核心概念

### 什么是优化器？

优化器就是 **用梯度更新权重的策略**。

第 4 章我们学了一个最简单的优化器：随机梯度下降（SGD）。

```
权重 = 权重 - 学习率 × 梯度
```

这就像蒙眼下山：感觉哪个方向最陡，就往那走一步。

但 SGD 有个大问题：**如果山路很曲折（梯度方向频繁变化），SGD 会走很多冤枉路**。

## 原理解析

### 1. SGD（随机梯度下降）

```python
class SGD:
    def __init__(self, lr=0.01):
        self.lr = lr

    def update(self, params, grads):
        for key in params.keys():
            params[key] -= self.lr * grads[key]
```

**类比：** 你蒙着眼睛下山，每次觉得往哪走最陡，就往那迈一步。

**缺点：** 如果地形很复杂（比如山谷），SGD 会来回震荡，走得慢。

### 2. Momentum（动量）

```python
class Momentum:
    def __init__(self, lr=0.01, momentum=0.9):
        self.lr = lr
        self.momentum = momentum
        self.v = None

    def update(self, params, grads):
        if self.v is None:
            self.v = {}
            for key in params.keys():
                self.v[key] = np.zeros_like(params[key])

        for key in params.keys():
            self.v[key] = self.momentum * self.v[key] - self.lr * grads[key]
            params[key] += self.v[key]
```

**类比：** 下山时你有"惯性"。上次往东走了 10 步，这次即使梯度说往东北，你还是会偏东一点。

**好处：** 在梯度方向一致时加速，在梯度频繁变化时抑制震荡。

### 3. AdaGrad（自适应学习率）

```python
class AdaGrad:
    def __init__(self, lr=0.01):
        self.lr = lr
        self.h = None

    def update(self, params, grads):
        if self.h is None:
            self.h = {}
            for key in params.keys():
                self.h[key] = np.zeros_like(params[key])

        for key in params.keys():
            self.h[key] += grads[key] ** 2
            params[key] -= self.lr * grads[key] / (np.sqrt(self.h[key]) + 1e-7)
```

**类比：** 每个参数有自己的"学习率"。更新频繁的参数学习率越来越小，更新少的参数学习率保持较大。

**好处：** 适合稀疏数据（比如 NLP 中有些词出现频繁，有些很少见）。

### 4. Adam（自适应矩估计）

Adam 是 Momentum 和 AdaGrad 的结合，是目前最常用的优化器。

```python
class Adam:
    def __init__(self, lr=0.001, beta1=0.9, beta2=0.999):
        self.lr = lr
        self.beta1 = beta1
        self.beta2 = beta2
        self.iter = 0
        self.m = None
        self.v = None

    def update(self, params, grads):
        if self.m is None:
            self.m, self.v = {}, {}
            for key in params.keys():
                self.m[key] = np.zeros_like(params[key])
                self.v[key] = np.zeros_like(params[key])

        self.iter += 1
        lr_t = self.lr * np.sqrt(1.0 - self.beta2 ** self.iter) / (1.0 - self.beta1 ** self.iter)

        for key in params.keys():
            self.m[key] += (1 - self.beta1) * (grads[key] - self.m[key])
            self.v[key] += (1 - self.beta2) * (grads[key] ** 2 - self.v[key])
            params[key] -= lr_t * self.m[key] / (np.sqrt(self.v[key]) + 1e-7)
```

## 避坑指南

1. **SGD 不是不好，而是慢** - 对于简单问题，SGD 最终也能找到好结果，只是需要更多时间。
2. **Adam 也不是万能的** - 有些研究证明 SGD 在某些情况下泛化能力更好。
3. **学习率仍然是最重要的超参数** - 无论用哪个优化器，学习率设错了都不行。
