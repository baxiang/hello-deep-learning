# 8.2 超参数调优

## 学习目标

- 理解什么是超参数
- 掌握超参数的搜索方法
- 学会用交叉验证评估模型

## 核心概念

### 什么是超参数？

超参数是 **我们在训练之前需要手动设置的参数**，不是网络自己学出来的。

常见的超参数：
- 学习率
- 批次大小（Batch Size）
- 网络层数
- 每层的节点数
- 权重衰减系数
- Dropout 比例

**类比：** 超参数就像烤蛋糕时的温度和时间。蛋糕配方（网络结构）定了，但烤多久、多少度需要你试。

## 原理解析

### 超参数搜索的策略

1. **随机搜索（推荐）** - 在合理范围内随机选值，比网格搜索更高效。
2. **网格搜索** - 把所有可能的组合都试一遍。参数多时非常慢。
3. **贝叶斯优化** - 根据之前的结果智能地选下一个尝试值。更高级但实现复杂。

### 超参数的范围建议

| 超参数 | 推荐范围 | 说明 |
|--------|---------|------|
| 学习率 | 10⁻⁶ ~ 10⁻² | 以对数尺度搜索 |
| 批次大小 | 16 ~ 256 | 2 的幂次（32, 64, 128） |
| 权重衰减 | 10⁻⁸ ~ 10⁻⁴ | 通常很小 |
| Dropout | 0 ~ 0.5 | 0 表示不用 |

### 用验证集评估

不要用测试集调超参数！应该把训练数据分成两部分：
- **训练集** - 用来训练网络。
- **验证集** - 用来调超参数。

最后用测试集做最终评估。

## 代码实战

```python
import numpy as np

def train_and_evaluate(lr, weight_decay, batch_size, dropout_ratio):
    """模拟训练和评估（实际应该跑完整的训练循环）"""
    # 这里用随机模拟：超参数越好，"准确率"越高
    # 实际使用时替换为真实的训练代码
    score = 0.5
    score += np.random.normal(0, 0.05)  # 随机波动
    
    # 学习率在 0.001~0.01 之间时比较好
    if 0.001 <= lr <= 0.01:
        score += 0.1
    
    # 权重衰减太小容易过拟合
    if 1e-6 <= weight_decay <= 1e-4:
        score += 0.05
    
    # 批次大小适中比较好
    if 32 <= batch_size <= 128:
        score += 0.05
    
    # Dropout 在 0.2~0.4 之间比较好
    if 0.2 <= dropout_ratio <= 0.4:
        score += 0.05
    
    return min(score, 0.99)  # 最高 99%

# 随机搜索超参数
if __name__ == "__main__":
    np.random.seed(42)
    
    best_score = 0
    best_params = {}
    
    print("开始超参数随机搜索...\n")
    
    for trial in range(10):
        # 在对数尺度上随机采样
        lr = 10 ** np.random.uniform(-6, -2)
        weight_decay = 10 ** np.random.uniform(-8, -4)
        batch_size = int(2 ** np.random.uniform(4, 8))  # 16 ~ 256
        dropout_ratio = np.random.uniform(0, 0.5)
        
        score = train_and_evaluate(lr, weight_decay, batch_size, dropout_ratio)
        
        print(f"试验 {trial+1}:")
        print(f"  学习率: {lr:.6f}, 权重衰减: {weight_decay:.2e}, 批次大小: {batch_size}, Dropout: {dropout_ratio:.2f}")
        print(f"  验证准确率: {score:.4f}")
        
        if score > best_score:
            best_score = score
            best_params = {
                'lr': lr,
                'weight_decay': weight_decay,
                'batch_size': batch_size,
                'dropout_ratio': dropout_ratio
            }
    
    print(f"\n最佳结果: 准确率 {best_score:.4f}")
    print(f"最佳超参数: {best_params}")
```

**逐行解析：**

1. `def train_and_evaluate(...)` - 定义一个函数，模拟训练和评估过程。实际使用时应该替换为真实的训练循环。
2. `score = 0.5` - 初始准确率设为 50%（随机猜测的水平）。
3. `score += np.random.normal(0, 0.05)` - 加入随机波动，模拟训练中的不确定性。
4. `if 0.001 <= lr <= 0.01: score += 0.1` - 学习率在这个范围内时，模型表现更好，加分。
5. `return min(score, 0.99)` - 限制最高准确率为 99%，防止超过合理范围。
6. `lr = 10 ** np.random.uniform(-6, -2)` - **关键行**：在对数尺度上采样学习率，这样 10⁻⁶ 和 10⁻² 被选中的概率相等。
7. `batch_size = int(2 ** np.random.uniform(4, 8))` - 批次大小在 2⁴=16 到 2⁸=256 之间随机选。
8. `score = train_and_evaluate(...)` - 用当前超参数跑一次训练，得到验证集上的准确率。
9. `if score > best_score:` - 如果这次结果比之前都好，就记录下来。
10. `best_params = {...}` - 保存当前最好的超参数组合。

## 避坑指南

1. **不要用测试集调参** - 如果用测试集调超参数，测试集就失去了"独立评估"的意义。
2. **搜索次数** - 随机搜索通常 50-100 次就够了。太少找不到好参数，太多浪费时间。
3. **记录每次结果** - 一定要保存每次试验的超参数和结果，方便分析和复现。
