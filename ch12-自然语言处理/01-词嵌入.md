# 12.1 词嵌入

## 学习目标

- 理解词嵌入的核心思想
- 掌握 Word2Vec 的两种方法
- 了解词向量的性质和应用

## 核心概念

### 词嵌入：把词变成向量

**计算机不理解文字，只理解数字。**

最早的方法：one-hot 编码
```
"猫" → [0, 0, 1, 0, 0, ...]  # 很长的向量，只有一个是1
"狗" → [0, 1, 0, 0, 0, ...]  # 和"猫"完全不同
```

问题：
- 向量太长（词汇表有多少词就多长）
- "猫"和"狗"的距离和"猫"和"汽车"一样（都是正交）
- 没有语义信息

**词嵌入：用短向量表示词，相似词向量相近。**

```
"猫" → [0.2, 0.8, 0.1, ...]  # 短向量
"狗" → [0.3, 0.7, 0.2, ...]  # 和"猫"接近！
"汽车" → [0.9, 0.1, 0.5, ...] # 和"猫"、"狗"都远
```

词嵌入让计算机"理解"词的语义。

### Word2Vec：学习词向量

Word2Vec 是最著名的词嵌入方法，有两种：

| 方法 | 核心思想 | 特点 |
|------|----------|------|
| Skip-gram | 用中心词预测周围词 | 适合小数据，学到更多语义 |
| CBOW | 用周围词预测中心词 | 速度快，适合大数据 |

**Skip-gram：**
```
句子："我喜欢深度学习"
中心词："喜欢"
预测："我"、"深度"（周围的词）
```

**CBOW：**
```
句子："我喜欢深度学习"
周围词："我"、"深度"
预测："喜欢"（中心词）
```

### 词向量的神奇性质

**相似词向量相近：**
```
向量("猫") 和向量("狗") 的距离很小
向量("猫") 和向量("汽车") 的距离很大
```

**可以做"词运算"：**
```
向量("国王") -向量("男") +向量("女") ≈ 向量("女王")
```

**类比推理：**
```
中国 : 北京 = 日本 : ?
→ 向量("中国") -向量("北京") +向量("日本") ≈向量("东京")
```

## 原理解析

### Skip-gram 的训练

目标：最大化预测周围词的概率。

```
P(周围词 | 中心词) = softmax(中心词向量 · 周围词向量)
```

训练过程：
1. 选一个中心词
2. 预测周围的词（窗口大小内的词）
3. 如果预测错了，调整词向量
4. 重复多次，词向量逐渐学到语义

### 为什么能学到语义？

**核心假设：出现在相似上下文的词，语义相似。**

```
"我喜欢猫"
"我喜欢狗"
```

"猫"和"狗"都出现在"我喜欢__"这个上下文，所以它们的向量会接近。

这就是**分布式语义假设**：词的含义由它的上下文决定。

### 词向量的维度

常用维度：100-300

- 太小（如10）：语义信息不够丰富
- 太大（如1000）：计算开销大，可能过拟合

经验：300 维是常用选择。

## 代码实战

### 词向量操作演示

```python
import numpy as np

def cosine_similarity(v1, v2):
    """计算两个向量的余弦相似度"""
    return np.dot(v1, v2) / (np.linalg.norm(v1) * np.linalg.norm(v2))

def find_similar(word, word_vectors, top_k=3):
    """找最相似的词"""
    target_vec = word_vectors[word]
    similarities = {}

    for w, vec in word_vectors.items():
        if w != word:
            similarities[w] = cosine_similarity(target_vec, vec)

    # 排序返回最相似的
    sorted_words = sorted(similarities.items(), key=lambda x: x[1], reverse=True)
    return sorted_words[:top_k]

# === 模拟词向量 ===
np.random.seed(42)

# 模拟词向量（实际应用用预训练的）
word_vectors = {
    "猫": np.array([0.2, 0.8, 0.1, 0.3]),
    "狗": np.array([0.3, 0.7, 0.2, 0.4]),  # 和猫接近
    "汽车": np.array([0.9, 0.1, 0.5, 0.2]),  # 和猫、狗都远
    "老虎": np.array([0.25, 0.75, 0.15, 0.35]),  # 和猫接近（都是动物）
    "自行车": np.array([0.85, 0.15, 0.45, 0.25]),  # 和汽车接近
}

print("=== 词嵌入演示 ===")

# 计算相似度
print("\n词向量相似度:")
print(f"猫 vs狗: {cosine_similarity(word_vectors['猫'], word_vectors['狗']):.3f}")
print(f"猫 vs 汽车: {cosine_similarity(word_vectors['猫'], word_vectors['汽车']):.3f}")
print(f"猫 vs 老虎: {cosine_similarity(word_vectors['猫'], word_vectors['老虎']):.3f}")
print(f"汽车 vs 自行车: {cosine_similarity(word_vectors['汽车'], word_vectors['自行车']):.3f}")

# 找最相似的词
print("\n找最相似的词:")
print(f"'猫' 最相似的词: {find_similar('猫', word_vectors)}")
print(f"'汽车' 最相似的词: {find_similar('汽车', word_vectors)}")

# 词运算（模拟）
print("\n词运算演示:")
print("国王 - 男 + 女 ≈ 女王")
print("（实际需要更大的词向量库才能看到效果）")

print("\n=== 词嵌入的关键 ===")
print("1. 词向量让计算机'理解'语义")
print("2. 相似词的向量接近")
print("3. 可以做词运算和类比推理")
print("4. 核心思想：上下文相似的词，语义相似")
```

## 避坑指南

1. **词汇表外的词**：预训练词向量可能不包含某些词，需要特殊处理。

2. **多义词问题**：一个词可能有多个含义（"苹果"可以是水果或公司），单一向量无法区分。

3. **维度选择**：不是越大越好，要根据任务和数据量选择。
