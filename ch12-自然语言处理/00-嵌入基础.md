# 12.0 嵌入基础

## 学习目标

- 理解嵌入的核心思想：字符到向量
- 掌握嵌入向量的训练过程
- 了解嵌入矩阵的作用
- 明白向量如何输入神经网络

## 开篇问题

**我们已经学会用编号表示字符：'a'=1, 'b'=2。**

问题来了：**编号有语义信息吗？**

先看答案：编号只是数字，没有语义。嵌入向量才有语义信息。

本章会用具体的数值演示嵌入如何工作。

## 核心概念

### 1. 简单编号的问题

**编号方法：给每个字符分配一个数字。**

```
简单编号：
  'a' = 1
  'b' = 2
  'c' = 3
  ...
  'z' = 26
```

**编号的缺陷：**

```
缺陷1：没有语义信息

  'a' = 1
  'b' = 2
  
  问题：
    模型只知道'a'是数字1
    不知道'a'和'b'都是字母
    不知道它们有什么关系
  
类比：
  给你两个学生的编号：1号、2号
  你不知道1号和2号是男生还是女生
  编号只是标签，没有信息

缺陷2：无法捕捉关系

  '猫'（拼音）和'狗'（拼音）
  
  如果编号：
    '猫' = 100
    '狗' = 200
  
  问题：
    |100 - 200| = 100
    |'猫' - '汽车'|（编号假设为300）= 200
    
    差距都是数字差，没有语义意义
  
  结果：
    '猫'和'狗'的距离
    '猫'和'汽车'的距离
    对模型来说都是"数字差"，没有区别

缺陷3：无法表达相似性

  'king'和'queen'应该相似（都是皇室）
  
  如果编号：
    'k' = 11
    'q' = 17
  
  问题：
    |11 - 17| = 6
    模型不理解它们相似
  
  结果：
    编号方法无法让模型"理解"语义关系
```

### 2. 嵌入向量：用一组数字表示字符

**嵌入的核心：一个字符用一组数字（向量）表示。**

```
编号方法：
  'a' = 1（一个数字）
  
嵌入方法：
  'a' = [0.2, 0.5, 0.1, 0.3]（一组数字，向量）

向量定义：
  向量是一个有序的数字集合
  
  例子：
    'a' 的向量 = [0.2, 0.5, 0.1, 0.3]
    
    第1位：0.2
    第2位：0.5
    第3位：0.1
    第4位：0.3
    
关键：
  每个数字的位置是固定的
  交换位置会得到不同的向量
  
  [0.2, 0.5, 0.1, 0.3] ≠ [0.5, 0.2, 0.3, 0.1]
```

**嵌入向量的优势：**

```
优势1：有语义信息

  'a' = [0.2, 0.5, 0.1, 0.3]
  
  假设：
    第1位：表示"是否是字母"（0.2 → 是）
    第2位：表示"是否是元音"（0.5 → 是）
    第3位：表示"使用频率"（0.1 → 低频）
    第4位：其他特征
  
  结果：
    模型能从向量读取信息
    不只是编号标签

优势2：相似字符向量接近

  '猫' = [0.8, 0.2, 0.5, 0.1]
  '狗' = [0.7, 0.3, 0.4, 0.2]
  
  距离计算：
    欧氏距离 = √((0.8-0.7)² + (0.2-0.3)² + ...)
    ≈ 0.22（很小）
  
  '汽车' = [0.1, 0.9, 0.8, 0.5]
  
  距离：
    '猫' vs '汽车' ≈ 1.2（很大）
  
  结果：
    '猫'和'狗'距离小 → 相似
    '猫'和'汽车'距离大 → 不相似
  
  模型理解语义关系！

优势3：可以捕捉复杂关系

  'king' 和 'queen' 的向量
  
  假设：
    'king' = [0.9, 0.1, 0.5, 0.3]
    'queen' = [0.85, 0.15, 0.5, 0.4]
  
  特征解读：
    第1位：皇室（0.9和0.85都很高）
    第2位：性别（0.1=男，0.15=女）
    第3位：权威（都是0.5）
  
  结果：
    向量表达复杂关系
    编号无法做到
```

### 3. 嵌入向量如何训练？

**嵌入向量不是人工设定的，是模型训练出来的。**

```
训练过程（类似权重训练）：

步骤1：初始化随机向量
  
  'a' = [随机数, 随机数, 随机数, 随机数]
  'b' = [随机数, 随机数, 随机数, 随机数]
  
  初始时：
    向量是随机的，没有意义

步骤2：输入到神经网络
  
  输入句子："cat"
  输入向量：'c'向量 + 'a'向量 + 't'向量
  
步骤3：计算损失
  
  神经网络输出预测
  与真实结果对比 → 计算损失

步骤4：调整向量（梯度下降）
  
  如果预测错误：
    调整权重
    也调整嵌入向量
  
  目标：
    让模型预测更准
    向量逐渐学到语义

步骤5：多轮迭代
  
  经过大量训练：
    '猫'的向量 → 学到"动物"特征
    '狗'的向量 → 学到"动物"特征
    相似字符 → 向量接近
```

**数值演示：嵌入向量训练过程**

```
训练任务：预测下一个字符

=== 初始阶段 ===

初始化向量（随机）：
  'a' = [0.5, 0.3, 0.1, 0.7]
  'b' = [0.2, 0.9, 0.4, 0.1]
  'c' = [0.8, 0.1, 0.6, 0.2]

训练数据：
  输入："ab"
  真实下一个字符：'c'

神经网络：
  输入：'a'向量 + 'b'向量
  输出：预测下一个字符
  
  假设预测：
    'c'概率 = 0.3（较低）
    'd'概率 = 0.5（错误地高）
  
  损失：较高

反向传播：
  调整权重
  调整向量：
    'a'向量 → 增加"与c相关"特征
    'b'向量 → 增加"与c相关"特征

=== 多轮训练后 ===

向量变化：
  'a' = [0.5, 0.3, 0.8, 0.7]  ← 第3位增加（与c相关）
  'b' = [0.2, 0.9, 0.8, 0.1]  ← 第3位增加
  'c' = [0.8, 0.1, 0.6, 0.2]

新训练数据：
  输入："bc"
  真实下一个字符：'d'
  
  'b'和'c'向量 → 都学到"后面常是d"

=== 大量训练后 ===

最终向量：
  'a' → 学到语义特征
  'b' → 学到语义特征
  'c' → 学到语义特征
  
特点：
  相似字符 → 向量接近
  模型理解语义关系
```

### 4. 嵌入矩阵：方便管理向量

**嵌入矩阵 = 所有字符向量的表格。**

```
假设：
  26个字母
  每个字母10维向量

嵌入矩阵：
  大小：10 × 26
  
  结构：
    列1：'a'的向量
    列2：'b'的向量
    列3：'c'的向量
    ...
    列26：'z'的向量
  
  每列是一个字符的向量
  每行是向量的一个维度
```

**数值演示：嵌入矩阵结构**

```
嵌入矩阵（简化，假设3维向量，3个字母）：

字母：'a', 'b', 'c'
向量维度：3

矩阵：
        'a'   'b'   'c'
维度1  [0.2, 0.3, 0.8]
维度2  [0.5, 0.1, 0.4]
维度3  [0.1, 0.9, 0.6]

每列是向量：
  'a'向量 = [0.2, 0.5, 0.1]（第1列）
  'b'向量 = [0.3, 0.1, 0.9]（第2列）
  'c'向量 = [0.8, 0.4, 0.6]（第3列）

查找方法：
  要找'a'的向量 → 看第1列
  要找'b'的向量 → 看第2列
```

**如何使用嵌入矩阵？**

```
输入句子："cat"

步骤1：找到每个字母的编号
  'c' = 3
  'a' = 1
  't' = 20

步骤2：查找嵌入矩阵
  'c'向量 → 第3列：[0.8, 0.4, 0.6]
  'a'向量 → 第1列：[0.2, 0.5, 0.1]
  't'向量 → 第20列：[...]

步骤3：拼接向量输入神经网络
  输入层：所有向量拼接
```

### 5. 向量如何输入神经网络？

**输入：多个字符 → 拼接多个向量。**

```
假设：
  每个字符用10维向量
  输入句子："humpty dumpt"（12个字符）

向量拼接：
  'h'向量：[10个数字]
  'u'向量：[10个数字]
  'm'向量：[10个数字]
  ...
  
拼接结果：
  总长度 = 12字符 × 10数字 = 120个数字

输入层：
  需要120个神经元
  每个神经元接收一个数字
```

**数值演示：向量拼接输入**

```
输入："humpty dumpt"（12字符，为简化假设2维向量）

字符向量（假设）：
  'h' = [0.5, 0.2]
  'u' = [0.3, 0.8]
  'm' = [0.7, 0.1]
  'p' = [0.4, 0.6]
  't' = [0.9, 0.3]
  'y' = [0.2, 0.7]
  ' ' = [0.1, 0.1]
  'd' = [0.6, 0.4]
  'u' = [0.3, 0.8]
  'm' = [0.7, 0.1]
  'p' = [0.4, 0.6]
  't' = [0.9, 0.3]

拼接向量：
  [0.5, 0.2, 0.3, 0.8, 0.7, 0.1, 0.4, 0.6, 
   0.9, 0.3, 0.2, 0.7, 0.1, 0.1, 0.6, 0.4,
   0.3, 0.8, 0.7, 0.1, 0.4, 0.6, 0.9, 0.3]
  
总长度：12 × 2 = 24个数字

输入层神经元：
  需要24个神经元
  神经元1：接收0.5（'h'向量第1维）
  神经元2：接收0.2（'h'向量第2维）
  神经元3：接收0.3（'u'向量第1维）
  ...
  神经元24：接收0.3（'t'向量第2维）
```

**为什么向量长度必须相同？**

```
问题：如果不同字符向量长度不同

  'h'向量：[0.5, 0.2]（2维）
  'u'向量：[0.3, 0.8, 0.1]（3维）
  
无法拼接：
  神经元3接收什么？'u'的第1维还是'h'的第3维？
  混乱，无法处理

解决：
  所有字符向量长度必须相同
  
  例如：
    所有字母都用10维向量
    所有单词都用300维向量
    
结果：
  拼接后长度 = 字符数 × 向量维度
  固定，可以输入神经网络
```

## 原理解析

### 嵌入训练的本质

**嵌入向量通过"预测任务"学习语义。**

```
训练任务：预测下一个字符

训练数据：
  "The cat sat on the mat"
  
训练过程：
  输入："The cat sa"
  碚实下一个：'t'
  
  模型预测：
    如果向量正确 → 预测't'概率高
    如果向量错误 → 预测't'概率低
  
  反向传播：
    调整向量，让预测更准
  
多轮训练：
  'cat'向量 → 学到"后面常是sat"
  'sat'向量 → 学到"后面常是on"
  'mat'向量 → 学到"与cat相似"（都是动物相关）

结果：
  相似上下文的词 → 向量接近
  模型学到语义关系
```

### 分布式语义假设

**核心假设：出现在相似上下文的词，语义相似。**

```
句子：
  "The cat sat on the mat"
  "The dog sat on the rug"

观察：
  'cat' 和 'dog' 都出现在 "The ___ sat on the" 上下文
  
假设：
  上下文相似 → 语义相似
  
训练结果：
  'cat'向量 ≈ 'dog'向量
  'mat'向量 ≈ 'rug'向量
  
模型理解：
  cat和dog都是动物
  mat和rug都是地毯
```

### 嵌入矩阵的优势

```
优势1：统一管理
  
  26字母 → 一张矩阵
  10k词汇 → 一张矩阵
  
  不需要单独存储每个向量

优势2：快速查找
  
  要找'a'向量 → 看第1列
  要找'b'向量 → 看第2列
  
  直接索引，速度快

优势3：可以复用
  
  训练好的嵌入矩阵：
    可以用在多个模型
    可以用在多个任务
  
  例如：
    训练好的英语字母嵌入
    可以用在英语生成模型
    可以用在英语分类模型
```

## 代码实战

### 嵌入矩阵演示

```python
import numpy as np

class SimpleEmbedding:
    """简化的嵌入矩阵演示"""
    
    def __init__(self, vocab_size=26, embed_dim=10):
        self.vocab_size = vocab_size  # 词汇表大小（26字母）
        self.embed_dim = embed_dim    # 向量维度
        
        # 初始化嵌入矩阵（随机）
        self.embedding_matrix = np.random.randn(embed_dim, vocab_size) * 0.1
        
        # 字符编号映射（a=0, b=1, ..., z=25）
        self.char_to_idx = {chr(ord('a') + i): i for i in range(vocab_size)}
    
    def get_vector(self, char):
        """查找字符的向量"""
        idx = self.char_to_idx[char]
        return self.embedding_matrix[:, idx]  # 返回第idx列
    
    def encode_sentence(self, sentence):
        """编码句子：拼接所有字符向量"""
        vectors = []
        for char in sentence.lower():
            if char in self.char_to_idx:
                vectors.append(self.get_vector(char))
        
        # 拼接向量
        if vectors:
            return np.concatenate(vectors)
        else:
            return np.array([])
    
    def similarity(self, char1, char2):
        """计算两个字符向量的相似度"""
        v1 = self.get_vector(char1)
        v2 = self.get_vector(char2)
        
        # 余弦相似度
        return np.dot(v1, v2) / (np.linalg.norm(v1) * np.linalg.norm(v2))

# 测试嵌入矩阵
embed = SimpleEmbedding(vocab_size=26, embed_dim=10)

print("=== 嵌入矩阵演示 ===")
print(f"词汇表大小：{embed.vocab_size}（26字母）")
print(f"向量维度：{embed.embed_dim}")
print(f"嵌入矩阵大小：{embed.embedding_matrix.shape}（10×26）")
print()

# 查找字符向量
print("查找字符向量：")
print("'a'向量（前5维）：", embed.get_vector('a')[:5])
print("'b'向量（前5维）：", embed.get_vector('b')[:5])
print("'c'向量（前5维）：", embed.get_vector('c')[:5])
print()

# 编码句子
sentence = "cat"
encoded = embed.encode_sentence(sentence)
print(f"编码句子 '{sentence}'：")
print(f"拼接向量长度：{len(encoded)}（3字符×10维=30）")
print(f"向量（前10维）：{encoded[:10]}")
print()

# 计算相似度
print("字符相似度（随机初始化，可能不相似）：")
print(f"'a' vs 'b': {embed.similarity('a', 'b'):.3f}")
print(f"'c' vs 'd': {embed.similarity('c', 'd'):.3f}")
print("（训练后，相似字符会接近）")
```

def vector_concat_demo():
    """向量拼接演示"""
    
    # 模拟字符向量（简化，假设2维）
    char_vectors = {
        'h': np.array([0.5, 0.2]),
        'u': np.array([0.3, 0.8]),

        'p': np.array([0.4, 0.6]),
        't': np.array([0.9, 0.3]),
    }
    
    sentence = "humpt"
    
    print("=== 向量拼接演示 ===")
    print(f"句子：'{sentence}'")
    print(f"字符数：{len(sentence)}")
    print(f"向量维度：2")
    print()
    
    # 显示每个字符向量
    print("每个字符向量：")
    for char in sentence:
        print(f"'{char}': {char_vectors[char]}")
    print()
    
    # 拼接向量
    concatenated = np.concatenate([char_vectors[c] for c in sentence])
    
    print("拼接后的向量：")
    print(concatenated)
    print(f"总长度：{len(concatenated)}（{len(sentence)}字符×2维）")
    print()
    
    print("输入层神经元：")
    for i, value in enumerate(concatenated):
        char_idx = i // 2
        dim_idx = i % 2
        char = sentence[char_idx]
        print(f"神经元{i+1}：{value:.1f}（'{char}'向量第{dim_idx+1}维）")

# 测试向量拼接
vector_concat_demo()

## 避坑指南

1. **嵌入向量维度选择**
   - 太小（如2维）：语义信息不足
   - 太大（如1000维）：计算成本高，可能过拟合
   - 建议：字符10-50维，单词100-300维

2. **所有向量长度必须相同**
   - 不同长度 → 无法拼接 → 无法输入网络
   - 解决：统一维度

3. **嵌入矩阵可以复用**
   - 训练好的嵌入 → 用在多个任务
   - 建议：使用预训练嵌入（如Word2Vec、GloVe）

4. **嵌入不只是字符级**
   - 字符嵌入：'a', 'b', 'c'
   - 单词嵌入：'cat', 'dog'
   - 子词嵌入：'cat', 's'
   - 建议：根据任务选择合适的嵌入单位

5. **嵌入向量捕捉相似性**
   - 训练后，相似字符向量接近
   - 但需要大量训练数据
   - 建议：用预训练嵌入，比自己训练更好

## 总结

**嵌入的本质：用向量表示字符，向量有语义信息。**

```
编号 vs 嵌入对比：

|方法 | 表示 | 语义信息 | 相似性 |
|------|------|----------|--------|
| 编号 | 单数字 | 无 | 无 |
| 嵌入 | 向量 | 有 | 有 |

编号：
  'a' = 1
  → 模型只知道数字1
  → 不知道语义

嵌入：
  'a' = [0.2, 0.5, 0.1, 0.3]
  → 模型能读取特征
  → 知道语义关系
```

**嵌入训练的核心：**

```
分布式语义假设：
  相似上下文的词 → 语义相似 → 向量接近

训练过程：
  初始化随机向量
  通过预测任务训练
  调整向量，让预测更准
  最终学到语义特征
```

**嵌入矩阵的作用：**

```
统一管理所有向量：
  26字母 → 10×26矩阵
  每列是一个字符向量

  'c'向量（10） + 'a'向量（10） + 't'向量（10）

输入层：
  30个神经元
  接收拼接后的向量
```
