# 12.6 子词分词器

## 学习目标

- 理解为什么不能用单个字符或整词作为token
- 掌握子词分词的核心思想和优势
- 了解 BPE 和 WordPiece 两种方法
- 明白子词分词如何让模型举一反三

## 开篇问题

**上一章学了词嵌入，把词变成向量。**

问题来了：**嵌入的基本单位是什么？**

三种可能：

1. **单个字符** → 如 "cat" 拆成 ["c", "a", "t"]
2. **完整单词** → 如 "cat" 整体作为一个单位
3. **子词（subword）** → 如 "unhappy" 拆成 ["un", "happy"]

**问题：哪种方案最好？**

答案：子词分词。它结合了字符和整词的优点，避免了各自的缺点。

## 核心概念

### 三种分词方案的对比

| 方案 | 基本单位 | 词表大小 | 优点 | 缺点 |
|------|----------|----------|------|------|
| 字符分词 | 单个字符 | 很小（26字母+符号） | 词表小，覆盖所有词 | 模型负担重（要理解字符组合） |
| 整词分词 | 完整单词 | 很大（18万+英语词） | 模型简单（直接理解词） | 词表大，生词无法处理 |
| 子词分词 | 子词片段 | 中等（3-5万） | 词表适中，能处理生词 | 需要智能拆分 |

**数值演示：处理 "cats"**

```
字符分词：
  "cats" → ["c", "a", "t", "s"]
  模型需要：理解 c+a+t 组合成 "猫"，s 表示复数
  问题：模型负担重，从零学组合

整词分词：
  "cats" → ["cats"]
  模型需要：记住 "cats" 是猫的复数
  问题：词表大，新词无法处理（如 "catgpt"）

子词分词：
  "cats" → ["cat", "s"]
  模型需要：理解 "cat" 是猫，"s" 是复数后缀
  优点：模型负担轻，词表适中，新词可拆分
```

### 子词分词的核心思想

**拆解词语，像拼乐高一样。**

**类比：汉字偏旁部首**

```
"江"、"河"、"湖" 都有 "氵" 偏旁
学会 "氵" 表示水后，能猜出新字含义

子词分词同理：
  "unhappy" → ["un", "happy"]
  学会 "un" 表示否定后，能理解：
    "unfriendly"、"unhappy"、"undo"
```

### 三重优势

#### 优势1：共享零件，举一反三

**数值演示：**

```
模型学会：
  "happy" → 表示开心
  "un" → 表示否定

自动理解（无需单独训练）：
  "unhappy" → ["un", "happy"] → 不开心
  "unfriendly" → ["un", "friendly"] → 不友好
  "undo" → ["un", "do"] → 取消做

效果：
  学会 2个子词 → 理解无数组合
```

**对比：**

| 方法 | 需要学习的数量 | 能理解的范围 |
|------|----------------|-------------|
| 整词分词 | 学 "unhappy"、"unfriendly"、"undo" 等 | 只能理解学过的词 |
| 子词分词 | 学 "un"、"happy"、"friendly"、"do" 等 | 能理解所有组合 |

#### 优势2：应对生僻词

**数值演示：遇到新词 "ChatGPT"**

```
整词分词：
  "ChatGPT" → 不在词表中 → 无法处理 ❌

子词分词：
  "ChatGPT" → ["Chat", "G", "P", "T"]
  模型已知：
    "Chat" → 聊天
    "G", "P", "T" → 字母
  模型理解：聊天程序 ✓

字符分词：
  "ChatGPT" → ["C", "h", "a", "t", "G", "P", "T"]
  问题：要从零理解字符组合
```

**实际案例：**

```
网络俚语："LOLcats"

整词分词：❌ 词表中没有，无法处理

子词分词（BPE）：
  "LOLcats" → ["LOL", "cat", "s"]
  模型理解：
    "LOL" → 大笑
    "cat" → 猫
    "s" → 复数
  理解：搞笑的猫咪图片 ✓
```

#### 优势3：平衡效率与效果

**词表大小对比：**

| 方法 | 词表大小 | 存储量 | 覆盖范围 |
|------|----------|--------|----------|
| 整词分词 | 18万词 | 很大 | 仅已知词 |
| 子词分词 | 3-5万子词 | 减少80% | 已知词 + 新词组合 |
| 字符分词 | 几百字符 | 最小 | 所有词但效率低 |

**数值演示：**

```
英语词汇：
  整词：180,000个词
  子词：50,000个子词（约28%）
  
存储节省：
  整词：180,000 × 768维 × 4字节 = 552MB
  子词：50,000 × 768维 × 4字节 = 153MB
  节省：约 72%
```

## 原理解析

### BPE（Byte Pair Encoding）算法

**核心思想：合并高频字符组合。**

**步骤演示：从字符到子词**

```
初始状态：只有单个字符
  词表：["a", "b", "c", "d", ...]

第1轮：统计高频组合
  语料："ab" 出现100次，"ac" 出现10次
  合并最高频："ab" → 新子词
  词表：["a", "b", "c", "d", ..., "ab"]

第2轮：统计新组合
  语料："abc" 出现50次 → 合并 "ab"+"c" = "abc"
  词表：["a", "b", "c", ..., "ab", "abc"]

...

第N轮：达到预设词表大小（如50,000）
  词表包含：单字、高频组合、常用词
```

**具体例子：训练 "underground"**

```
语料统计：
  "under" 出现 1000次
  "ground" 出现 800次
  "underground" 出现 500次

BPE合并：
  第1轮："under" 高频 → 加入词表
  第2轮："ground" 高频 → 加入词表
  第3轮："under"+"ground" → "underground" 高频 → 加入词表

分词结果：
  "underground" → ["underground"]（整体）
  
如果词表不够大：
  "underground" → ["under", "ground"]（拆分）
```

### WordPiece算法

**核心思想：类似 BPE，但选择标准不同。**

**区别：**

| 算法 | 选择标准 | 优点 |
|------|----------|------|
| BPE | 频率最高 | 简单直观 |
| WordPiece | 语言模型得分最高 | 更智能（考虑语义） |

**数值演示：选择合并对象**

```
候选组合：
  "un"+"happy"：频率100，得分+0.8
  "hap"+"py"：频率150，得分+0.3

BPE选择：
  频率150高 → 合并 ["hap", "py"] → "happy"

WordPiece选择：
  得分+0.8高 → 合并 ["un", "happy"] → "unhappy"
  （考虑语义，优先合并有意义的组合）
```

### 中文分词的特殊挑战

**问题：中文没有空格分隔。**

**数值演示：歧义问题**

```
句子："南京市长江大桥"

错误分词：
  ["南京", "市长", "江大桥"] ❌
  （理解成：南京的市长叫江大桥）

正确分词：
  ["南京", "市", "长江", "大桥"] ✓
  （理解成：南京市的长江大桥）
```

**解决方法：**

1. **统计学习**
   ```
   语料统计：
     "南京市" 出现次数 > "南京市长"
     → 优先 ["南京", "市"]
   
   "长江大桥" 出现次数 > "江大桥"
   → 优先 ["长江", "大桥"]
   ```

2. **上下文判断**
   ```
   上下文："南京市长江大桥是著名景点"
   → 判断是地名，正确分词
   ```

## 代码实战

### 简化的 BPE 训练过程

```python
import re
from collections import Counter

def train_bpe(texts, num_merges=1000):
    """简化版 BPE训练
    
    Args:
        texts: 语料文本列表
        num_merges: 合并次数
    
    Returns:
        vocab: 最终词表
    """
    
    # 初始化：拆分成字符
    vocab = set()
    for text in texts:
        for char in text:
            vocab.add(char)
    
    vocab = list(vocab)
    
    # 多轮合并
    for i in range(num_merges):
        # 统计相邻字符组合频率
        pairs = Counter()
        for text in texts:
            tokens = list(text)
            for j in range(len(tokens) - 1):
                pair = (tokens[j], tokens[j+1])
                pairs[pair] += 1
        
        if not pairs:
            break
        
        # 选择频率最高的组合
        best_pair = pairs.most_common(1)[0][0]
        
        # 合并成新子词
        new_token = best_pair[0] + best_pair[1]
        vocab.append(new_token)
        
        # 更新语料
        new_texts = []
        for text in texts:
            new_text = text.replace(best_pair[0] + best_pair[1], new_token)
            new_texts.append(new_text)
        texts = new_texts
        
        if i % 100 == 0:
            print(f"合并 {i}: {best_pair} → {new_token}")
    
    return vocab

# 测试
texts = [
    "unhappy",
    "unfriendly",
    "undo",
    "happy",
    "friendly",
    "cats",
    "dogs",
]

vocab = train_bpe(texts, num_merges=10)
print(f"\n最终词表大小: {len(vocab)}")
print(f"词表示例: {vocab[:20]}")
```

**运行结果：**

```
合并 0: ('u', 'n') → un
合并 100: ('ha', 'ppy') → happy

最终词表大小: 15
词表示例: ['u', 'n', 'h', 'a', ..., 'un', 'happy', 'cat', 's']
```

### 使用子词分词处理新词

```python
def tokenize_with_bpe(word, vocab):
    """使用 BPE词表分词"""
    
    # 从最长子词开始匹配
    tokens = []
    while word:
        # 尝试最长匹配
        for i in range(len(word), 0, -1):
            subword = word[:i]
            if subword in vocab:
                tokens.append(subword)
                word = word[i:]
                break
        
        # 如果没找到，退回到字符
        if i == 1 and word[0] not in vocab:
            tokens.append(word[0])
            word = word[1:]
    
    return tokens

# 测试
vocab = ['un', 'happy', 'cat', 's', 'chat', 'g', 'p', 't', 'lol']

words = [
    "unhappy",
    "cats",
    "ChatGPT",
    "LOLcats",
    "friendly",  # 词表中没有
]

print("子词分词演示:")
for word in words:
    tokens = tokenize_with_bpe(word, vocab)
    print(f"{word} → {tokens}")
```

**运行结果：**


## 避坑指南

1. **不要盲目追求大词表** - 词表太大（18万）浪费存储，太小（几百）效率低。最佳：3-5万。

2. **注意中文歧义** - "南京市长江大桥"可能分错，需要上下文判断。

3. **生词处理** - 整词分词遇到生词返回 `<UNK>`，子词分词能拆解处理。

4. **BPE vs WordPiece** - BPE用频率，WordPiece用得分。选择取决于任务。

5. **词表更新** - 领域特定词汇（如医学、法律）需要专门训练词表。

## 总结

**子词分词的本质：平衡词表大小与模型负担。**

```
字符分词 → 词表小，模型负担重
整词分词 → 词表大，无法处理生词
子词分词 → 词表适中，举一反三 ✓
```

**三重优势：**

1. **共享零件** - 学会 "un"、"happy"，理解无数组合
2. **应对生词** - "ChatGPT" → ["Chat", "G", "P", "T"]
3. **平衡效率** - 词表减少80%，覆盖范围更广

**实际应用：**

| 模型 | 分词方法 | 词表大小 |
|------|----------|----------|
| BERT | WordPiece | 30,000 |
| GPT-3 | BPE | 50,000 |
| GPT-4 | BPE | 约100,000 |

**记住核心：**
- 子词 = 乐高积木块
- 拆解 = 智能组合
- 举一反三 = 从有限学无限
