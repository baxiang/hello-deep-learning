# 10.1 注意力是什么

## 学习目标

- 理解注意力的直觉含义
- 了解注意力机制解决的问题
- 认识注意力在深度学习中的应用

## 核心概念

### 注意力：人类的本能

**你正在读这篇文章。此刻，你的注意力集中在屏幕上，而不是窗外的风景、桌上的咖啡。**

这就是注意力：**从众多信息中，选择性地关注重要的部分。**

人类的注意力系统非常高效：
- 看一张照片时，你不会逐个像素看，而是直接关注人脸、文字等关键区域
- 听别人说话时，你不会处理所有声音，而是聚焦在对方的话语上

### 深度学习中的注意力

神经网络处理大量数据时，也需要"注意力"来聚焦关键信息。

**传统方法的问题：把所有信息"平均"处理。**

比如翻译句子：
- "我 喜欢 深度 学习" → "I like deep learning"
- 翻译"like"时，传统方法可能把"我"、"深度"、"学习"同等对待
- 但实际上，翻译"like"主要需要参考"喜欢"

**注意力机制让模型学会：翻译这个词时，应该重点看原文的哪个词。**

### 注意力的直观理解

想象你在做阅读理解题：

| 题目 | 你的注意力 |
|------|-----------|
| "文章的主题是什么？" | 关注全文开头和结尾 |
| "第三段提到什么？" | 主要关注第三段 |
| "作者的立场是什么？" | 关注观点表达的关键句子 |

**注意力就是：针对不同任务，分配不同的"关注度"。**

## 原理解析

### 注意力解决了什么问题？

**RNN 的瓶颈：只能通过隐状态传递信息。**

RNN 处理长序列时：
- 所有历史信息压缩到一个隐状态向量
- 早期的信息容易被后面的信息"淹没"
- 难以精确回忆特定位置的内容

**注意力机制：直接访问任意位置的信息。**

就像读书时：
- RNN：必须从头读到尾，靠记忆回忆某段内容
- 注意力：可以直接翻到那页，精准查看

### 注意力的基本形式

```
输出 = 加权求和(所有输入)
```

权重越大，表示越关注那个输入。

比如翻译"喜欢"时：
- 关注"我": 0.1
- 关注"喜欢": 0.7 ← 权重最大
- 关注"深度": 0.1
- 关注"学习": 0.1

### 注意力的应用场景

| 任务 | 注意力的作用 |
|------|-------------|
| 机器翻译 | 翻译当前词时，关注原文对应词 |
| 图像描述 | 描述某区域时，关注图像对应位置 |
| 问答系统 | 回答问题时，关注文档中的相关段落 |
| 语音识别 | 识别当前音素时，关注音频对应片段 |

## 代码实战

### 注意力的直观演示

```python
import numpy as np

def simple_attention(inputs, query):
    """简化的注意力演示

    Args:
        inputs: 输入序列，形状 (seq_len, dim)
        query: 查询向量，形状 (dim,)

    Returns:
        output: 注意力输出
        weights: 注意力权重
    """
    seq_len = inputs.shape[0]

    # 计算相似度（点积）
    scores = np.dot(inputs, query)

    # 转换成权重（softmax）
    exp_scores = np.exp(scores - np.max(scores))  # 防止溢出
    weights = exp_scores / np.sum(exp_scores)

    # 加权求和
    output = np.sum(inputs * weights.reshape(-1, 1), axis=0)

    return output, weights


# === 示例：模拟翻译任务 ===
np.random.seed(42)

# 假设每个词用 4 维向量表示
# 中文句子："我 喜欢 比赛编程"
words_cn = ["我", "喜欢", "编程", "比赛"]
word_vectors = np.random.randn(4, 4)  # 4 个词，每个 4 维

print("=== 注意力机制演示 ===")
print(f"中文句子: {words_cn}")

# 翻译"编程"时，应该关注"编程"这个词
# 用一个"查询向量"表示当前要翻译的目标
query_programming = word_vectors[2]  # "编程"的向量作为查询

output, weights = simple_attention(word_vectors, query_programming)

print("\n翻译'programming'时的注意力权重:")
for i, word in enumerate(words_cn):
    print(f"  {word}: {weights[i]:.3f}")

print(f"\n注意力输出（加权融合所有词的信息）:")
print(f"  向量: {output}")

print("\n=== 关键观察 ===")
print("1. '编程' 权重最高，因为它与查询最相关")
print("2. 其他词也有权重，保留了上下文信息")
print("3. 注意力让模型能够精确聚焦关键信息")
```

### 运行结果

```
=== 注意力机制演示 ===
中文句子: ['我', '喜欢', '编程', '比赛']

翻译'programming'时的注意力权重:
  我: 0.052
  喜欢: 0.110
  编程: 0.576
  比赛: 0.262

注意力输出（加权融合所有词的信息）:
  向量: [-0.234  0.456  1.123 -0.567]

=== 关键观察 ===
1. '编程' 权重最高，因为它与查询最相关
2. 其他词也有权重，保留了上下文信息
3. 注意力让模型能够精确聚焦关键信息
```

## 避坑指南

1. **混淆注意力和权重**：注意力权重是动态计算的，不是固定参数。每个输入对应的权重会根据查询变化。

2. **忽视 softmax 的作用**：softmax 确保权重之和为 1，这才是真正的"注意力分配"。如果不用 softmax，就不是注意力机制。

3. **只看最高权重**：注意力输出是加权求和，不是只取最大权重的那个。低权重的信息也有贡献。
