# 10.4 多头注意力

## 学习目标

- 理解多头注意力的设计思想
- 掌握多头注意力的计算过程
- 了解"多头"类比的实际意义

## 核心概念

### 多头注意力：多角度看问题

**单头注意力：只用一种方式看句子。**
**多头注意力：用多种方式看句子。**

类比：分析一篇文章时，你可能会从多个角度思考：
- **语法角度**：关注句子结构、主谓宾关系
- **语义角度**：关注词语含义、上下文联系
- **引用角度**：关注"他"、"这个"指的是什么
- **位置角度**：关注前后顺序关系

多头注意力就像有多个"专家"，每个专家关注不同的侧面。

### 多头的优势

| 特点 | 说明 |
|------|------|
| 多视角 | 不同的头关注不同的关系 |
| 并行计算 | 各头独立计算，可以并行 |
| 信息丰富 | 融合多种视角的信息 |
| 表达能力强 | 单头可能遗漏某些关系 |

### 多头注意力的结构

```
输入 X → 分成 h 个头
        ↓
    head_1: Q_1, K_1, V_1 → Attention_1 → output_1
    head_2: Q_2, K_2, V_2 → Attention_2 → output_2
    ...
    head_h: Q_h, K_h, V_h → Attention_h → output_h
        ↓
    拼接所有output → Concat → 线性变换 → 最终输出
```

每个头有自己的 QKV 权重，学习不同的注意力模式。

## 原理解析

### 多头注意力的计算

**1. 分头**

把输入 X 投影到多个子空间：

```
Q_i = X · W_q_i    # 第 i 个头的 Q
K_i = X · W_k_i    # 第 i 个头的 K
V_i = X · W_v_i    # 第 i 个头的 V
```

每个头的维度是 d_model / h。

**2. 各头独立计算注意力**

```
head_i = Attention(Q_i, K_i, V_i)
```

**3. 拼接并融合**

```
MultiHead = Concat(head_1, ..., head_h) · W_o
```

W_o 把所有头的信息融合起来。

### 多头注意力计算：完整数值演示

**例子：句子 "我喜欢深度学习"，4个头，16维向量**

```
配置：
  输入：4个词
  向量维度：16维
  头数量：4个
  每头维度：16 ÷ 4 = 4维
```

**步骤1：每个头有独立的投影矩阵**

```
为什么每个头能看到不同信息？
  关键：每个头有自己的 Wq、Wk、Wv 矩阵

类比：戴着不同颜色眼镜看同一幅画

  红1镜片（头1）：只看到红色元素（语法特征）
  蓝镜片（头2）：只看到蓝色元素（长距离依赖）
  绿镜片（头3）：只看到绿色元素（语义关系）
  黄镜片（头4）：只看到黄色元素（引用关系）

输入相同（同一幅画），但各头看到不同
```

**步骤2：分割维度（不增加计算量）**

```
不增加计算负担的设计：

原始方案（计算量翻倍）：
  4个头 × 16维向量 = 64维（计算量爆炸）✗

分割方案（计算量不变）：
  把16维切成4份 → 每份4维 ✓
  
数值演示：
  
输入向量（'我'）：
  [v1, v2, v3, v4, v5, v6, v7, v8, v9, v10, v11, v12, v13, v14, v15, v16]
  
分割成4份：
  头1：[v1, v2, v3, v4]（前4维）
  头2：[v5, v6, v7, v8]（第5-8维）
  头3：[v9, v10, v11, v12]（第9-12维）
  头4：[v13, v14, v15, v16]（第13-16维）

结果：
  每个头处理4维 → 总计算量不变
  但4个头看不同子空间 → 信息丰富
```

**步骤3：各头独立计算注意力**

```
假设：
  头1（语法专家）：关注主谓关系
  头2（长距离专家）：关注远距离依赖
  头3（语义专家）：关注词义相似
  头4（引用专家）：关注"它"指代什么

数值演示（简化，假设1维）：

头1（语法）对 "我" 的注意力：
  '我' vs '喜欢'：权重0.7（主谓关系强）
  '我' vs '深度'：权重0.1（无关）
  '我' vs '学习'：权重0.2（弱相关）

头2（长距离）对 "我" 的注意力：
  '我' vs '喜欢'：权重0.2
  '我' vs '深度'：权重0.3
  '我' vs '学习'：权重0.5（远距离依赖）

头3（语义）对 "我" 的注意力：
  '我' vs '喜欢'：权重0.4
  '我' vs '深度'：权重0.5（语义相关）
  '我' vs '学习'：权重0.1

头4（引用）对 "我" 的注意力：
  '我' vs '喜欢'：权重0.3
  '我' vs '深度'：权重0.2
  '我' vs '学习'：权重0.5（可能被引用）

结果：
  不同头有不同的关注重点
  融合多角度信息
```

**步骤4：拼接所有头的输出**

```
各头输出（假设）：

头1输出（'我'）：[a1]（1维，语法特征）
头2输出（'我'）：[a2]（1维，长距离特征）
头3输出（'我'）：[a3]（1维，语义特征）
头4输出（'我'）：[a4]（1维，引用特征）

拼接：
  [a1, a2, a3, a4]（4维）

完整例子（每头4维）：
  头1输出：[0.5, 0.3, 0.2, 0.8]
  头2输出：[0.1, 0.6, 0.4, 0.2]
  头3输出：[0.7, 0.2, 0.9, 0.3]
  头4输出：[0.3, 0.5, 0.1, 0.6]
  
拼接后：
  [0.5, 0.3, 0.2, 0.8, 0.1, 0.6, 0.4, 0.2, 0.7, 0.2, 0.9, 0.3, 0.3, 0.5, 0.1, 0.6]
  
长度：4+4+4+4 = 16维（恢复原维度）
```

**步骤5：融合层（Wo矩阵）**

```
拼接后的向量：16维
需要融合各头信息：线性变换

Wo矩阵：16×16
拼接向量 × Wo → 最终输出（16维）

作用：
  把4个头的信息融合
  相当于"综合4个专家意见"
```

**完整流程总结：**

```
输入：'我'（16维向量）

步骤1：分割维度
  16维 → 4份 × 4维
  头1：前4维
  头2：第5-8维
  头3：第9-12维
  头4：第13-16维

步骤2：各头独立计算
  头1：语法角度 → 输出4维
  头2：长距离角度 → 输出4维
  头3：语义角度 → 输出4维
  头4：引用角度 → 输出4维

步骤3：拼接
  4维+4维+4维+4维 = 16维

步骤4：融合
  16维 × Wo → 最终输出16维

结果：
  '我'的新向量融合了4个角度信息
  比单头更丰富、更全面
```

### 多头注意力的威力

**威力1：远近兼顾**

```
短程关注：
  头1专门盯着相邻词
  
  例子：
    "在那遥远的" → 关注后面"地方"
    相邻词关系强 → 捕捉短语

长程关注：
  头2专门盯着远距离词
  
  例子：
    文章开头："张三去了北京"
    文章结尾："他回到了上海"
  
  头2连接"张三"和"他"（长距离引用）
  
结果：
  不"读了后面忘前面"
  记住远距离依赖
```

**威力2：特征分离（专家分工）**

```
训练后的自发分工：

头1：语法专家
  关注动词和宾语
  "我 喜欢 学习" → 主谓宾关系

头2：翻译专家（机器翻译任务）
  关注源语言和目标语言对应词
  "apple" ↔ "苹果"

头3：语义专家
  关注词义相似
  "猫" ↔ "狗"（都是动物）

头4：引用专家
  关注指代关系
  "张三" ↔ "他"

结果：
  不同专家分工
  模型更强大
```

**威力3：增强鲁棒性**

```
容错能力：

单头注意力：
  如果一个头坏了 → 全部信息丢失
  
多头注意力：
  头1坏了 → 还有头2、头3、头4
  部分"专家"失效 → 其他"专家"补偿
  
类比：
  医院有多个专科医生
  一个医生请假 → 其他医生替代
  
结果：
  更鲁棒、更稳定
```

### 为什么这样设计？

**不同头学习不同的模式：**

实验发现，多头注意力中：
- 有的头主要关注相邻词（局部依赖）
- 有的头主要关注句子两端（全局依赖）
- 有的头主要关注动词和主语
- 有的头主要关注形容词和名词

就像团队分工：每个成员负责一个方面，最后汇总。

### 头数的选择

Transformer 论文用 8 个头。

经验：
- 头太少（1-2）：表达能力不足
- 头太多（16+）：计算开销大，可能过拟合
- 通常 4-8 个头效果不错

## 代码实战

```python
import numpy as np


def softmax(x, axis=-1):
    """Softmax 函数"""
    e_x = np.exp(x - np.max(x, axis=axis, keepdims=True))
    return e_x / np.sum(e_x, axis=axis, keepdims=True)


class MultiHeadAttention:
    """多头注意力模块"""

    def __init__(self, d_model, num_heads):
        """初始化

        Args:
            d_model: 模型维度
            num_heads: 头的数量
        """
        assert d_model % num_heads == 0, "d_model 必须能被 num_heads 整除"

        self.d_model = d_model
        self.num_heads = num_heads
        self.d_k = d_model // num_heads  # 每个头的维度

        # 初始化各头的 QKV 权重
        np.random.seed(42)
        self.W_q = np.random.randn(d_model, d_model) * 0.1
        self.W_k = np.random.randn(d_model, d_model) * 0.1
        self.W_v = np.random.randn(d_model, d_model) * 0.1
        self.W_o = np.random.randn(d_model, d_model) * 0.1

    def forward(self, X):
        """前向传播

        Args:
            X: 输入序列，形状 (seq_len, d_model)

        Returns:
            output: 多头注意力输出
            all_weights: 所有头的注意力权重
        """
        seq_len = X.shape[0]

        # 计算 QKV
        Q = np.dot(X, self.W_q)  # (seq_len, d_model)
        K = np.dot(X, self.W_k)
        V = np.dot(X, self.W_v)

        # 分头：reshape 成 (num_heads, seq_len, d_k)
        Q_heads = Q.reshape(seq_len, self.num_heads, self.d_k).transpose(1, 0, 2)
        K_heads = K.reshape(seq_len, self.num_heads, self.d_k).transpose(1, 0, 2)
        V_heads = V.reshape(seq_len, self.num_heads, self.d_k).transpose(1, 0, 2)

        # 各头计算注意力
        all_weights = []
        all_outputs = []

        for h in range(self.num_heads):
            # 计算分数
            scores = np.dot(Q_heads[h], K_heads[h].T) / np.sqrt(self.d_k)

            # softmax
            weights = softmax(scores)
            all_weights.append(weights)

            # 加权求和
            head_output = np.dot(weights, V_heads[h])
            all_outputs.append(head_output)

        # 拼接所有头: (seq_len, d_model)
        concat = np.concatenate(all_outputs, axis=-1)

        # 最终线性变换
        output = np.dot(concat, self.W_o)

        return output, np.array(all_weights)


if __name__ == "__main__":
    print("=" * 50)
    print("多头注意力演示")
    print("=" * 50)

    np.random.seed(42)

    # 模拟句子
    seq_len = 4
    d_model = 16
    num_heads = 4

    words = ["我", "喜欢", "深度", "学习"]
    X = np.random.randn(seq_len, d_model)

    print(f"\n配置:")
    print(f"  模型维度: {d_model}")
    print(f"  头数量: {num_heads}")
    print(f"  每头维度: {d_model // num_heads}")

    # 多头注意力
    mha = MultiHeadAttention(d_model, num_heads)
    output, all_weights = mha.forward(X)

    print(f"\n输入形状: {X.shape}")
    print(f"输出形状: {output.shape}")
    print(f"权重矩阵数量: {len(all_weights)}")

    # 显示每个头的注意力
    print("\n=== 各头的注意力权重 ===")
    for h in range(num_heads):
        print(f"\n头 {h} 的注意力矩阵:")

    print("\n=== 各头关注模式分析 ===")
    for h in range(num_heads):
