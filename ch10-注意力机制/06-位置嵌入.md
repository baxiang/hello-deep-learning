# 10.6 位置嵌入

## 学习目标

- 理解为什么 Transformer 需要位置信息
- 掌握三种位置编码方法：Sin/Cos、可学习、RoPE
- 明白位置嵌入如何与词嵌入结合
- 能用代码实现 Sin/Cos 位置编码

## 开篇问题

**上一章学了多头注意力，它能关注句子中任意两个词的关系。**

问题来了：**Transformer 能感知词的顺序吗？**

答案：不能！Transformer 有个"盲点"——对顺序不敏感。

**数值演示：**

```
句子1："小明打小红"
句子2："小红打小明"

Transformer 看到的：
  句子1：{小明, 打, 小红}
  句子2：{小红, 打, 小红}

两个集合一样！Transformer 无法区分。
```

但语义完全不同：
- 句子1：小明是施动者，小红是被打者
- 句子2：小红是施动者，小明是被打者

**问题：如何让 Transformer 知道顺序？**

答案：位置嵌入（给模型装"GPS"）。

## 核心概念

### Transformer 的盲点：无序集合

**类比：三张卡片**

```
你把"我"、"爱"、"你"写在三张牌上，按顺序交给 Transformer。

Transformer 看到：
  一堆牌："我、爱、你"都在这

你打乱顺序："你、爱、我"

Transformer 看到：
  还是同一堆牌："我、爱、你"都在这

顺序变了，但 Transformer 认为内容没变！
```

**为什么会这样？**

自注意力的计算：
```
注意力分数 = Query × Key

Query、Key 都是从词向量计算来的。
词向量不管位置，只管语义。

所以："我"在任何位置的 Key 都一样。
```

### 位置嵌入：给词贴"位置标签"

**解决思路：给每个词加上位置信息。**

```
词向量：表示语义（"苹果"是什么）
位置向量：表示位置（"苹果"在第几位）

组合向量：语义 + 位置
```

**数值演示：**

```
假设 3 维词向量：

E(苹果) = [0.05, -0.02, 0.01]  （语义）

位置编码：
  第1位：P(1) = [0.84, 0.01, 0.00]
  第2位：P(2) = [0.09, 0.14, 0.00]
  第3位：P(3) = [-0.76, 0.28, 0.00]

如果"苹果"在第3位：
  组合向量 = E + P(3)
            = [0.05-0.76, -0.02+0.28, 0.01+0.00]
            = [-0.71, 0.26, 0.01]

现在"苹果"既有语义信息，又有位置信息。
```

### 为什么不能简单编号 1,2,3？

**直觉想法：**
```
第1个词标1
第2个词标2
第10000个词标10000
```

**数值演示：这个做法有问题！**

```
词向量（语义）：E(苹果) = [0.05, -0.02, 0.01]
  数值范围：-0.1 到 0.1（很小）

简单编号：P(10000) = [10000, 0, 0]
  数值范围：10000（很大）

叠加后：E + P = [10000.05, -0.02, 0.01]

问题：
  语义 0.05 被 10000"淹没"了！
  模型只看到"这是第10000位"，看不到"这是苹果"
```

**正确做法：位置向量数值要和词向量相近。**

```
词向量：[0.05, -0.02, 0.01]（数值小）
位置向量：[0.20, -0.91, 0.03]（数值也小）

叠加后：[0.25, -0.93, 0.04]

这样语义和位置都保留了，不会"喧宾夺主"。
```

## 原理解析

### 方法一：Sin/Cos 函数编码（原始Transformer）

**核心思想：用固定函数生成位置"指纹"。**

**公式（不用背，理解意思）：**

```
PE(pos, 2i) = sin(pos / 10000^(2i/d))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d))

其中：
  pos：位置（0, 1, 2, ...）
  i：维度索引（0, 1, 2, ...）
  d：向量总维度
```

**数值演示：如何生成位置向量？**

假设维度 d=4，位置 pos=0：

```
第0维（偶数）：PE(0, 0) = sin(0 / 10000^0) = sin(0) = 0
第1维（奇数）：PE(0, 1) = cos(0 / 10000^0) = cos(0) = 1
第2维（偶数）：PE(0, 2) = sin(0 / 10000^0.5) = sin(0) = 0
第3维（奇数）：PE(0, 3) = cos(0 / 10000^0.5) = cos(0) = 1

位置0的向量：[0, 1, 0, 1]
```

位置 pos=1：

```
第0维：PE(1, 0) = sin(1) ≈ 0.84
第1维：PE(1, 1) = cos(1) ≈ 0.54
第2维：PE(1, 2) = sin(0.01) ≈ 0.01
第3维：PE(1, 3) = cos(0.01) ≈ 1.00

位置1的向量：[0.84, 0.54, 0.01, 1.00]
```

**关键特点：**

1. **数值范围：-1 到 1**（不会淹没语义）
2. **每个位置独特**（不同位置有不同指纹）
3. **可无限延伸**（是函数，有位置就能算）

### 方法二：可学习位置嵌入（BERT、早期GPT）

**核心思想：让模型自己学一套位置编码。**

**做法：**

```
准备512个"位置向量槽位"：

  P[1] = [随机向量1]
  P[2] = [随机向量2]
  ...
  P[512] = [随机向量512]

训练时，模型调整这些向量：

  哪种P[pos]效果好 → 往那个方向调整

最终，512个向量被训练成"最适合本模型的位置编码"。
```

**数值演示：**

```
初始：P[3] = [0.12, -0.03, 0.07]（随机）

训练后：P[3] = [0.15, -0.05, 0.09]（模型学出来的）

词向量：E(苹果) = [0.05, -0.02, 0.01]

叠加：E + P[3] = [0.20, -0.07, 0.10]
```

**优点缺点：**

|优点 |缺点 |
|------|------|
| 更贴合模型需要 | 槽位数量固定（如512） |
| 不用写公式 | 无法处理超长文本（>512） |
| 训练学出来 | 每个模型要重新训练 |

### 方法三：旋转位置编码（RoPE，现代主流）

**核心思想：不用"加"位置，而是"转"向量。**

**类比：旋转箭头**

```
词向量是一个有方向的箭头（二维平面）

第1个位置：把箭头逆时针转10度
第2个位置：把箭头逆时针转20度
第3个位置：把箭头逆时针转30度
...

位置信息变成"旋转角度"。
```

**数值演示：**

```
词向量（2维）：E = [0.05, -0.02]

原始方向：角度 θ ≈ -22度

位置1的旋转：转10度 → θ = -12度
新向量：E_rotated = [0.06, -0.01]

位置2的旋转：转20度 → θ = -2度
新向量：E_rotated = [0.07, 0.00]
```

**关键：相对位置自动显现**

```
词A在第10位：转了100度
词B在第12位：转了120度

模型计算相关性时：
  只看"差值20度"（相对距离）

不需要知道绝对位置（第10、第12）
只需要知道相对关系（相隔2个词）
```

**优点：**

| 优点 | 说明 |
|------|------|
| 自然捕捉相对位置 | 注意力计算只看角度差 |
| 可处理超长文本 | 无限旋转 |
| 数学优雅 | 用旋转代替加法 |

## 代码实战

### 实现 Sin/Cos 位置编码

```python
import numpy as np

def positional_encoding_sin_cos(max_len, d_model):
    """Sin/Cos 位置编码
    
    Args:
        max_len: 最大长度（如100）
        d_model: 向量维度（如512）
    
    Returns:
        PE矩阵：形状(max_len, d_model)
    """
    PE = np.zeros((max_len, d_model))
    
    for pos in range(max_len):
        for i in range(d_model):
            if i % 2 == 0:  # 偶数维度用sin
                PE[pos, i] = np.sin(pos / (10000 ** (2*i/d_model)))
            else:  # 奇数维度用cos
                PE[pos, i] = np.cos(pos / (10000 ** (2*(i-1)/d_model)))
    
    return PE

#生成位置编码
PE = positional_encoding_sin_cos(100, 4)  # 100个位置，4维向量

print("位置0的编码:", PE[0])
print("位置1的编码:", PE[1])
print("位置2的编码:", PE[2])

# 验证特点
print("\n数值范围:")
print("最小值:", PE.min())
print("最大值:", PE.max())
print("(应该在-1到1之间)")
```

**运行结果：**

```
位置0的编码: [0.1.0.1.]
位置1的编码: [0.84 0.54 0.01 1.00]
位置2的编码: [0.91 0.42 0.02 1.00]

数值范围:
最小值: -1.0
最大值: 1.0
(应该在-1到1之间)
```

### 位置编码 + 词嵌入

```python
import numpy as np

def add_positional_embedding(word_embedding, pos, PE):
    """词嵌入 + 位置编码"""
    
    position_embedding = PE[pos]
    
    # 相加
    combined = word_embedding + position_embedding
    
    return combined

#模拟词嵌入
word_embedding = np.array([0.05, -0.02, 0.01, 0.03])

# 生成位置编码（假设4维）
PE = positional_encoding_sin_cos(100, 4)

# "苹果"在第3位
combined = add_positional_embedding(word_embedding, 3, PE)

print("词嵌入（语义）:", word_embedding)
print("位置编码（第3位）:", PE[3])
print("组合向量:", combined)
print("\n语义 + 位置，两者都保留了")
```

**运行结果：**

```
词嵌入（语义）: [0.05, -0.02, 0.01, 0.03]
位置编码（第3位）: [0.14 0.99 0.03 1.00]
组合向量: [0.19, 0.97, 0.04, 1.03]

语义 + 位置，两者都保留了
```

### 可学习位置嵌入（简化版）

```python
import numpy as np

class LearnablePositionalEmbedding:
    """可学习位置嵌入"""
    
    def __init__(self, max_len, d_model):
        self.max_len = max_len
        self.d_model = d_model
        
        # 初始化：随机向量
        self.P = np.random.randn(max_len, d_model) * 0.1
    
    def get_embedding(self, pos):
        """获取某个位置的编码"""
        if pos >= self.max_len:
            raise ValueError(f"位置{pos}超出范围（最大{self.max_len})")
        return self.P[pos]
    
    def update(self, pos, gradient):
        """训练时更新（简化版）"""
        self.P[pos] -= gradient * 0.01  # 学习率0.01

# 创建可学习位置嵌入
learnable_PE = LearnablePositionalEmbedding(512, 4)

print("位置3的编码（初始随机）:", learnable_PE.get_embedding(3))

# 模拟训练更新
learnable_PE.update(3, np.array([0.1, -0.05, 0.02, -0.01]))
print("位置3的编码（训练后）:", learnable_PE.get_embedding(3))
```

## 避坑指南

1. **数值范围要匹配** - 位置编码数值要和词嵌入相近，否则会淹没语义。

2. **Sin/Cos 是固定的** - 不参与训练，所有模型都一样。

3. **可学习有上限** - 超过 max_len 就没有位置编码可用。

4. **RoPE 更现代** - 相对位置编码，适合长文本。

5. **不要忘记叠加** - 位置编码要和词嵌入相加（或相乘），不能单独使用。

## 总结

**位置嵌入的本质：给无序的词集合加上顺序信息。**

| 方法 | 特点 | 适用场景 |
|------|------|----------|
| Sin/Cos | 固定函数，可无限延伸 | 原始Transformer |
| 可学习 | 模型自己学，更贴合 | BERT、早期GPT |
| RoPE | 相对位置，数学优雅 |现代主流模型 |

**记住核心：**
- Transformer 无法感知顺序 → 需要位置嵌入
- 位置编码数值要小 → 不能淹没语义
- 三种方法各有优劣 → 根据任务选择

**类比总结：**
- Sin/Cos：固定的GPS坐标系统
- 可学习：定制化的GPS地图
- RoPE：相对距离的指南针
