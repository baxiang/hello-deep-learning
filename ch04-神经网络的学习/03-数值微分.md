# 4.3 数值微分

## 学习目标

- 理解梯度的概念
- 学会用数值方法计算梯度
- 理解梯度的方向和意义

## 核心概念

### 什么是梯度？

梯度告诉我们在每个参数处，**函数往哪个方向变化最快**。

**类比：** 你站在山坡上，梯度就是你脚下最陡的方向。如果顺着梯度的反方向走，你就能最快到达山脚（损失最小的地方）。

### 导数 vs 梯度

- **导数** - 针对一个变量的函数，告诉你这个变量变化时，函数值变化多少。
- **梯度** - 针对多个变量的函数，是每个变量导数组成的向量。

## 原理解析

### 数值微分的计算

用很小的变化量 h 来近似导数：

```
f'(x) ≈ (f(x + h) - f(x - h)) / (2h)
```

这叫 **中心差分**，比只用 `(f(x + h) - f(x)) / h` 更精确。

```python
def numerical_diff(f, x):
    """数值微分"""
    h = 1e-4
    return (f(x + h) - f(x - h)) / (2 * h)

def numerical_gradient(f, x):
    """梯度计算（多变量）"""
    grad = np.zeros_like(x)
    for i in range(x.size):
        tmp = x[i]
        # 计算第 i 个变量的偏导数
        x[i] = tmp + 1e-4
        fxh1 = f(x)
        x[i] = tmp - 1e-4
        fxh2 = f(x)
        grad[i] = (fxh1 - fxh2) / (2 * 1e-4)
        x[i] = tmp
    return grad
```

## 代码实战

```python
import numpy as np

def numerical_diff(f, x):
    """数值微分（单变量）"""
    h = 1e-4
    return (f(x + h) - f(x - h)) / (2 * h)

def numerical_gradient(f, x):
    """梯度计算（多变量）"""
    h = 1e-4
    grad = np.zeros_like(x)
    for i in range(x.size):
        tmp_val = x[i].copy()
        # x[i] + h
        x[i] = tmp_val + h
        fxh1 = f(x)
        # x[i] - h
        x[i] = tmp_val - h
        fxh2 = f(x)
        grad[i] = (fxh1 - fxh2) / (2 * h)
        x[i] = tmp_val
    return grad

# === 示例 1：单变量函数的导数 ===
def f_single(x):
    return x ** 2

print("=== 单变量导数 ===")
print("f(x) = x² 在 x=3 处的导数:", numerical_diff(f_single, 3))
print("理论值: 2x = 6")

# === 示例 2：多变量函数的梯度 ===
def f_multi(x):
    """f(x0, x1) = x0² + x1²"""
    return x[0]**2 + x[1]**2

print("\n=== 多变量梯度 ===")
x = np.array([3.0, 4.0])
grad = numerical_gradient(f_multi, x)
print("f(x0, x1) = x0² + x1² 在 (3, 4) 处的梯度:", grad)
print("理论值: [2×3, 2×4] = [6, 8]")
```

**代码解读：**
- `numerical_diff` 用中心差分公式计算单变量函数在某点的导数。
- `numerical_gradient` 对多变量函数，逐个变量计算偏导数，组成梯度向量。
- `tmp_val = x[i].copy()` 保存原始值，计算完后恢复，防止污染原数组。
- 示例 1 验证 `f(x) = x²` 在 x=3 处的导数接近 6。
- 示例 2 验证 `f(x0, x1) = x0² + x1²` 在 (3, 4) 处的梯度接近 [6, 8]。

## 避坑指南

1. **h 不能太小也不能太大** - 太小会有浮点精度问题，太大近似不精确。1e-4 是个经验值。
2. **数值微分很慢** - 有多少个参数就要计算多少次。100 万个参数就要算 100 万次。这就是为什么需要反向传播（第 5 章）。
3. **修改原数组** - `numerical_gradient` 会修改传入的 x 数组，所以计算完要恢复原值（`x[i] = tmp_val`）。
