# 4.3 数值微分

## 学习目标

- 理解"梯度"到底是什么
- 学会用代码计算梯度
- 理解梯度的方向代表什么

## 核心概念

### 什么是梯度？

梯度回答一个简单的问题：**如果我稍微改一下这个权重，损失会变大还是变小？**

- 梯度是正数 → 权重变大，损失变大 → 应该把权重变小
- 梯度是负数 → 权重变大，损失变小 → 应该把权重变大

**类比 1：爬山**

你站在山坡上，眼睛蒙着布。你想知道"往哪边走下坡最快"。

梯度就是告诉你："往东走 1 步，高度变化多少"。如果往东走 1 步下降了 2 米，说明东边是下坡方向。

**类比 2：调温度**

你洗澡时调水温。你把热水旋钮往右拧一点点，水是变热了还是变冷了？变了多少？

梯度就是："往右拧一点，温度变化多少"。

### 导数 vs 梯度

- **导数** = 一个变量变化时，函数值变化多少。
- **梯度** = 所有变量的导数组成的列表。

**类比：**
- 导数 = 只调热水旋钮，水温变化多少。
- 梯度 = [热水旋钮调一点变化多少，冷水旋钮调一点变化多少]。

## 原理解析

### 数值微分：用"试一试"的方法算梯度

我们不知道梯度的公式怎么办？**试一试就行。**

把权重稍微改大一点，算一次损失；把权重稍微改小一点，算一次损失。两次损失的差，就是梯度的近似值。

```
梯度 ≈ (权重 + h 时的损失 - 权重 - h 时的损失) / (2 × h)
```

h 是一个很小的数，比如 0.0001。

**这到底意味着什么？**

就像你调水温：
1. 把热水旋钮往右拧一点点，记下水温。
2. 把热水旋钮往左拧一点点，记下水温。
3. 两次水温的差，就是"热水旋钮的梯度"。

### 为什么叫"中心差分"？

因为我们是在权重的"两边"都试了（+h 和 -h），取中间的值。比只试一边（+h）更准确。

**类比：** 你想知道一个地方的坡度。
- 只往一个方向走 → 可能刚好踩到一块石头，测不准。
- 往两个方向都走 → 取平均，更准。

## 代码实战

### 完整可运行代码

```python
import numpy as np

# === 1. 定义一个简单的函数：f(x) = x² ===
def function_x_squared(x):
    """一个简单的函数，方便验证"""
    return x ** 2

# === 2. 数值微分（单变量）===
def numerical_diff(f, x):
    """计算函数 f 在 x 处的导数"""
    h = 1e-4  # 很小的变化量
    return (f(x + h) - f(x - h)) / (2 * h)

# === 3. 验证：f(x) = x² 在 x=3 处的导数 ===
# 手算结果：f'(x) = 2x，所以 f'(3) = 6
result = numerical_diff(function_x_squared, 3.0)
print(f"f(x) = x² 在 x=3 处的导数（数值微分）: {result:.6f}")
print(f"手算结果（2×3）: {2 * 3}")
print(f"误差: {abs(result - 6):.10f}")

# === 4. 梯度（多变量）===
def numerical_gradient(f, x):
    """计算函数 f 在 x 处的梯度（多变量）"""
    h = 1e-4
    grad = np.zeros_like(x)  # 创建和 x 形状一样的零数组
    
    for i in range(x.size):
        # 保存原始值
        tmp = x[i]
        
        # 计算 f(x + h)
        x[i] = tmp + h
        fxh1 = f(x)
        
        # 计算 f(x - h)
        x[i] = tmp - h
        fxh2 = f(x)
        
        # 计算梯度
        grad[i] = (fxh1 - fxh2) / (2 * h)
        
        # 恢复原始值
        x[i] = tmp
    
    return grad

# === 5. 测试：f(x0, x1) = x0² + x1² 在 (3, 4) 处的梯度 ===
def function_2d(x):
    """f(x0, x1) = x0² + x1²"""
    return x[0] ** 2 + x[1] ** 2

x = np.array([3.0, 4.0])
grad = numerical_gradient(function_2d, x)
print(f"\nf(x0,x1) = x0² + x1² 在 (3, 4) 处的梯度: {grad}")
print(f"手算结果: [2×3, 2×4] = [6, 8]")
```

### 运行结果

```
f(x) = x² 在 x=3 处的导数（数值微分）: 6.000000
手算结果（2×3）: 6
误差: 0.0000000050

f(x0,x1) = x0² + x1² 在 (3, 4) 处的梯度: [6. 8.]
手算结果: [2×3, 2×4] = [6, 8]
```

**解读：**

1. 数值微分和手算结果几乎一样（误差只有 0.000000005）。
2. 梯度 `[6, 8]` 告诉我们：
   - x0 的梯度是 6 → x0 变大，f 会变大
   - x1 的梯度是 8 → x1 变大，f 会变大
   - x1 的梯度更大 → x1 对 f 的影响更大

### 逐行解读

**单变量数值微分：**
```python
h = 1e-4
return (f(x + h) - f(x - h)) / (2 * h)
```
1. `h = 1e-4` 是一个很小的数（0.0001）。
2. 算 `f(x + h)` 和 `f(x - h)`。
3. 两次的差除以 `2h`，就是梯度的近似值。

**多变量梯度：**
```python
for i in range(x.size):
    tmp = x[i]
    x[i] = tmp + h
    fxh1 = f(x)
    x[i] = tmp - h
    fxh2 = f(x)
    grad[i] = (fxh1 - fxh2) / (2 * h)
    x[i] = tmp
```
1. 对每个变量，分别计算它变化时的影响。
2. 每次只改一个变量，其他不变。
3. 计算完后恢复原始值，不影响下一次循环。

## 避坑指南

1. **h 不能太小也不能太大** - 太小（比如 1e-10）会有浮点数精度问题；太大（比如 0.1）近似不精确。1e-4 是个经验值。

2. **数值微分很慢** - 有多少个变量就要算多少次函数。100 万个参数就要算 100 万次。这就是为什么第 5 章要学"反向传播"——一种快速算梯度的方法。

3. **别忘了恢复原始值** - 计算完一个变量的梯度后，必须把 x 恢复原样，否则会影响下一个变量的计算。
