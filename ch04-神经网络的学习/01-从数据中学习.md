# 4.1 从数据中学习

## 学习目标

- 理解"学习"在神经网络中到底是什么
- 理解训练数据和测试数据为什么分开
- 能说出学习的 5 个步骤

## 开篇问题

**上一章的神经网络，权重是"神奇地"得到的。**

比如我们说"假设已经训练好一个神经网络"，权重是 `[0.1, 0.3, 0.5]`。

问题是：**这些权重从哪里来？**

两种可能：
1. 人手动设 → 不现实，参数太多（几万到几亿）
2. 让网络自己找 → 这就是"学习"

**本章问题：如何让网络自己找到好的权重？**

答案：用数据"教"网络，通过"训练"找到权重。

## 核心概念

### 数学本质：学习就是找权重

**神经网络的全部参数：**
- 权重（Weights）：神经元连线上数字
- 偏置（Bias）：神经元的额外数值

**学习的本质：找到一组权重和偏置，让网络输出符合预期。**

数值演示：

假设识别"叶子"的任务：

```
期望输出：叶子概率 = 1（100%），花朵概率 = 0

当前权重（随机）：叶子概率 = 0.6，花朵概率 = 0.4

差距：期望 1，实际 0.6，差 0.4

目标：调整权重，让差距变小（从 0.4 → 0.2 → 0.1 → 0）
```

这就是"学习"：不断调整权重，缩小差距。

### 什么是"学习"？

上一章的神经网络，权重是随便设的。就像一个没学过的人来做题，全靠蒙。

**"学习"就是：让网络自己找到好的权重，而不是靠人手动设。**

**类比：** 学骑自行车。
- 一开始你不会骑，歪歪扭扭（权重不对）。
- 每次快要摔倒，你会调整身体姿势（更新权重）。
- 摔了几百次后，你学会了（权重对了）。

**关键：** 你不是在"背"怎么骑，而是在"调"身体。神经网络也是一样，不是在"背"答案，而是在"调"权重。

### 学习的 5 个步骤

网络学习就像打游戏过关，每一步都是一个回合：

```
第 1 步：拿一批题（Mini-batch）
第 2 步：做题（前向传播）
第 3 步：对答案，看错多少（计算损失）
第 4 步：分析错题，找到改进方向（计算梯度）
第 5 步：调整学习方法（更新权重）
```

然后回到第 1 步，重复几千次、几万次…… 直到错题越来越少。

**数值演示：一个具体的训练回合**

假设训练数据：识别叶子（标注为"叶子"）

**第 1 步：拿一批题**

```
从训练数据中随机选 10 张图片
这 10 张都是叶子（标签：叶子=1，花朵=0）
```

**第 2 步：做题**

```
输入第 1 张图片的 RGB 值：[181, 216, 210, 12.0]

网络计算：
  加权和 → [0.3, 0.7, 1.1]
  激活 → [0.57, 0.67, 0.75]
  输出 → [0.6, 0.4]

解读：
  叶子概率 = 0.6
  花朵概率 = 0.4
```

**第 3 步：对答案**

```
正确答案：叶子=1，花朵=0
网络输出：叶子=0.6，花朵=0.4

误差：
  叶子：期望 1，实际 0.6，差 0.4
  花朵：期望 0，实际 0.4，差 0.4

总误差 = 0.4 + 0.4 = 0.8（很大！）
```

**第 4 步：找改进方向**

```
问题：叶子权重太小，花朵权重太大

分析：
  增加"叶子"相关权重 → 叶子输出变大（0.6 → 0.7）
  减少"花朵"相关权重 → 花朵输出变小（0.4 → 0.3）
```

**第 5 步：调整权重**

```
调整后的权重：
  叶子权重：从 0.1 → 0.12（增加）
  花朵权重：从 0.2 → 0.18（减少）

再次测试：
  输出 → [0.65, 0.35]
  误差 = 0.35 + 0.35 = 0.7（变小了！）
```

**关键观察：**
- 初始误差大（0.8）
- 调整权重后误差变小（0.7）
- 重复多次，误差会越来越小

这就是"学习"的具体过程：通过数据，不断调整权重。

**这到底意味着什么？**

- **Mini-batch** = 每次只做 10 道题，而不是一次做完 60000 道题。这样做更快，效果更好。
- **前向传播** = 用当前的水平做题。
- **损失** = 错了多少题。
- **梯度** = 往哪个方向调整能提高分数。
- **更新权重** = 调整学习方法。

### 训练数据 vs 测试数据

**训练数据** = 练习题。网络用来学习的。

**测试数据** = 考试题。网络没见过的，用来检验是不是真学会了。

**为什么要分开？**

**类比：** 如果学生把练习册的答案背下来，练习题全对，但考试遇到新题就不会了。这叫"过拟合"——记住了答案，没学会方法。

## 原理解析

### 随机梯度下降法（SGD）

"随机"指的是：每次都**随机**选一小批数据来学习，而不是用全部数据。

**为什么不一次性用全部数据？**

1. **太慢** - 60000 张图片一起算，计算机要算很久。
2. **效果更好** - 随机选一小批，每次学习方向略有不同，反而更容易找到最优解。

**类比：** 你想找到山谷的最低点。
- 一次看全地图 → 慢，而且地图可能不准。
- 每次看脚下的路 → 快，而且走得稳。

### 什么是"损失"？

损失 = 网络预测和正确答案之间的差距。

- 损失大 = 网络很差，需要大幅调整。
- 损失小 = 网络不错，接近正确答案了。

**学习的目标就是让损失越来越小。**

**类比：** 损失就像考试分数（反过来的）。分数越低 = 错得越多 = 损失越大。你想让损失从 100 分降到 0 分。

## 代码实战

### 模拟学习流程

下面用代码模拟一个学习的简化流程：

```python
import numpy as np

# === 1. 准备数据 ===
# 模拟 100 个样本，每个样本 2 个特征
x_train = np.random.randn(100, 2)      # 训练输入
t_train = np.random.randint(0, 2, 100) # 训练标签（0 或 1）

print("训练数据:", x_train.shape)  # (100, 2) = 100 个样本，每个 2 个特征
print("训练标签:", t_train.shape)  # (100,) = 100 个标签

# === 2. 随机选 Mini-batch ===
batch_size = 10
# 从 0 到 99 中随机选 10 个索引
batch_mask = np.random.choice(100, batch_size)

# 用这些索引取出对应的样本
x_batch = x_train[batch_mask]  # 10 个样本
t_batch = t_train[batch_mask]  # 10 个标签

print("\nMini-batch 索引:", batch_mask)
print("Mini-batch 输入形状:", x_batch.shape)  # (10, 2)
print("Mini-batch 标签形状:", t_batch.shape)  # (10,)
```

### 运行结果

```
训练数据: (100, 2)
训练标签: (100,)

Mini-batch 索引: [85 32 17 63 41  7 28 99 55 12]
Mini-batch 输入形状: (10, 2)
Mini-batch 标签形状: (10,)
```

### 逐行解读

**准备数据：**
```python
x_train = np.random.randn(100, 2)
```
- 生成 100 行 2 列的随机数。
- 每行是一个样本，2 列是样本的两个特征。

**随机选 Mini-batch：**
```python
batch_mask = np.random.choice(100, batch_size)
```
- `np.random.choice(100, 10)` 从 0~99 中随机选 10 个不重复的索引。
- 这 10 个索引就是这一轮要学习的数据。

**取出数据：**
```python
x_batch = x_train[batch_mask]
```
- 用索引从 100 个样本中取出 10 个。

## 避坑指南

1. **Mini-batch 的大小** - 太小（比如 1）学习不稳定；太大（比如 60000）学习慢。常用 32、64、128。

2. **随机性** - `np.random.choice` 默认是有放回抽样（同一个索引可能被选多次）。实际中更常用无放回抽样，但效果差别不大。

3. **学习率** - 更新权重的步长。太大 → 步子迈太大，跳过最优值；太小 → 走得太慢。通常从 0.01 开始试。
