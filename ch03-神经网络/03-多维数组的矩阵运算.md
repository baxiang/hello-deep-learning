# 3.3 多维数组的矩阵运算

## 学习目标

- 理解为什么神经网络要用矩阵运算
- 掌握矩阵乘法的维度变化规律
- 能用手算推导任意一层的输出形状

## 核心概念

### 为什么用矩阵运算？

神经网络每一层都要做这样的计算：

```
输出 = 权重 × 输入 + 偏置
```

如果一个层有 100 个输入、50 个输出，那就要做 100×50 = 5000 次乘法和加法。

**不用矩阵运算：** 写 5000 次乘法，代码又长又慢。

**用矩阵运算：** 一行代码搞定，而且计算机底层优化过，快几十倍。

**类比：** 你要批改 100 份作业。
- for 循环 = 一份一份改
- 矩阵运算 = 把所有作业铺在桌上，一起改

### 矩阵乘法的"对接"规则

矩阵乘法有一个铁律：**左边矩阵的列数 = 右边矩阵的行数**。

```
A (2, 3) · B (3, 4) → 结果 (2, 4)
       ↑↑ 必须相等
```

**类比：** 就像拼乐高。左边积木的凸起数（列）必须等于右边积木的凹槽数（行），才能拼在一起。拼完后的新积木，高度是左边的高度（2），宽度是右边的宽度（4）。

**结果的形状规律：**

```
结果形状 = (左边的行数, 右边的列数)
```

### 神经网络中的维度变化

以 2→3→2 的网络为例：

```
输入 x:   (2,)      ← 2 个输入
权重 W1: (2, 3)     ← 2 行（对应 2 个输入），3 列（对应 3 个隐藏节点）
结果 a1: (3,)       ← 3 个隐藏节点的加权和
```

计算过程：
```
x (2,) · W1 (2, 3) → 结果 (3,)
```

**记忆技巧：** 结果的形状 = 右边矩阵的列数。

## 原理解析

### 矩阵乘法的直观理解

假设你开了一家奶茶店，有两种奶茶：

| 奶茶 | 茶叶（克） | 牛奶（毫升） | 糖（克） |
|------|----------|------------|---------|
| 珍珠奶茶 | 5 | 200 | 10 |
| 水果奶茶 | 3 | 150 | 15 |

现在有 3 个订单：
- 订单 1：2 杯珍珠奶茶，1 杯水果奶茶
- 订单 2：1 杯珍珠奶茶，3 杯水果奶茶

问：每个订单需要多少茶叶、牛奶、糖？

**用矩阵乘法：**

```
订单矩阵 (2, 2) × 配方矩阵 (2, 3) = 需求矩阵 (2, 3)

订单数      每种奶茶的原料
           茶  奶  糖
[2, 1]   [5  200 10]   [13  550  35]   ← 订单 1：13 克茶，550ml 奶，35 克糖
[1, 3] × [3  150 15] = [14  650  55]   ← 订单 2：14 克茶，650ml 奶，55 克糖
```

**这到底意味着什么？**

矩阵乘法就是"批量计算"。一行代码算出所有订单的原料需求，不需要一个个算。

### 神经网络中的矩阵运算

回到神经网络。一个 2→3→2 的网络：

```
输入 x = [1.0, 0.5]         ← 2 个输入数字

第 1 层权重 W1 = 
    [0.1, 0.3, 0.5]         ← 输入 1 到 3 个隐藏节点的权重
    [0.2, 0.4, 0.6]         ← 输入 2 到 3 个隐藏节点的权重

第 1 层偏置 b1 = [0.1, 0.2, 0.3]   ← 3 个隐藏节点的偏置
```

计算加权和：
```
a1 = x · W1 + b1
   = [1.0, 0.5] · [[0.1, 0.3, 0.5], [0.2, 0.4, 0.6]] + [0.1, 0.2, 0.3]
   = [0.2, 0.5, 0.8] + [0.1, 0.2, 0.3]
   = [0.3, 0.7, 1.1]
```

**逐元素计算过程（方便理解）：**
- 第 1 个隐藏节点：1.0×0.1 + 0.5×0.2 + 0.1 = 0.3
- 第 2 个隐藏节点：1.0×0.3 + 0.5×0.4 + 0.2 = 0.7
- 第 3 个隐藏节点：1.0×0.5 + 0.5×0.6 + 0.3 = 1.1

## 代码实战

### 完整可运行代码

```python
import numpy as np

# === 输入数据 ===
x = np.array([1.0, 0.5])
print("输入 x:", x)
print("x 的形状:", x.shape)

# === 第 1 层的权重和偏置 ===
W1 = np.array([
    [0.1, 0.3, 0.5],
    [0.2, 0.4, 0.6]
])
b1 = np.array([0.1, 0.2, 0.3])

print("\n权重 W1 的形状:", W1.shape)   # (2, 3)
print("偏置 b1 的形状:", b1.shape)     # (3,)

# === 第 1 层计算 ===
a1 = np.dot(x, W1) + b1
print("\n第 1 层加权和 a1:", a1)
print("a1 的形状:", a1.shape)          # (3,)

# === 第 2 层的权重和偏置 ===
W2 = np.array([
    [0.1, 0.4],
    [0.2, 0.5],
    [0.3, 0.6]
])
b2 = np.array([0.1, 0.2])

print("\n权重 W2 的形状:", W2.shape)   # (3, 2)
print("偏置 b2 的形状:", b2.shape)     # (2,)

# === 第 2 层计算 ===
a2 = np.dot(a1, W2) + b2
print("\n第 2 层加权和 a2:", a2)
print("a2 的形状:", a2.shape)          # (2,)
```

### 运行结果

```
输入 x: [1.  0.5]
x 的形状: (2,)

权重 W1 的形状: (2, 3)
偏置 b1 的形状: (3,)

第 1 层加权和 a1: [0.3 0.7 1.1]
a1 的形状: (3,)

权重 W2 的形状: (3, 2)
偏置 b2 的形状: (2,)

第 2 层加权和 a2: [0.52 1.29]
a2 的形状: (2,)
```

### 维度变化总结

```
x (2,) → W1 (2,3) → a1 (3,) → W2 (3,2) → a2 (2,)
```

**规律：** 每一层的输出形状 = 权重矩阵的列数 = 这一层的节点数。

## 避坑指南

1. **形状对不上** - 报错 `ValueError: shapes not aligned` 时，检查 `np.dot()` 两边：左边的列数必须等于右边的行数。

2. **偏置为什么能直接加？** - 偏置的形状是 `(3,)`，加权和的形状也是 `(3,)`。形状一样，直接加。这就是"广播"的威力。

3. **批次处理时的维度** - 如果一次处理 100 条数据，输入形状变成 `(100, 2)`。计算方式完全一样，结果变成 `(100, 3)`。多了一行"批次维度"，其他不变。
