# 3.4 手写数字识别推理

## 学习目标

- 理解"推理"是什么（给网络输入数据，让它输出答案）
- 理解 MNIST 数据集的格式
- 掌握批处理的概念
- 能用代码跑通一个识别手写数字的推理流程

## 核心概念

### 什么是"推理"？

推理就是：**给训练好的网络输入新数据，让它给出答案**。

**类比：** 推理就像考试。网络学完了知识（训练），现在给它一套新卷子（测试数据），看它能不能答对。

**2026 年的例子：**
- 你用手机拍一张手写数字的照片，AI 立刻告诉你"这是 7"→ 这就是推理。
- ChatGPT 回答你的问题 → 也是推理（网络已经训练好了，现在用它来回答新问题）。

### MNIST 数据集：深度学习的"Hello World"

MNIST 是一个手写数字图片库：
- 60000 张训练图片（用来教网络）
- 10000 张测试图片（用来考网络）
- 每张图片是 28×28 像素的灰度图
- 每张图里有一个手写数字（0-9）

**一张图片长什么样？**

28×28 像素 = 784 个格子，每个格子有一个灰度值（0=黑色，255=白色）。

```
28 列
┌────────────────────────────┐
│ ░░░▓▓▓░░░░░░░░░░░░░░░░░ │ ← 第 1 行
│ ░░▓▓▓▓▓░░░░░░░░░░░░░░░░ │
│ ░░▓▓░░▓▓░░░░░░░░░░░░░░░ │
│ ... 共 28 行 ...           │
└────────────────────────────┘
28 行
```

### 推理的步骤

1. **准备数据** - 把图片变成网络能吃的格式（784 个数字的一维数组）
2. **前向传播** - 把数据喂给网络，从输入层传到输出层
3. **判断结果** - 输出层有 10 个数字，最大的那个就是答案

**类比：**
1. 把照片扫描成数字信号
2. 送到 AI 大脑里处理
3. AI 告诉你"这是数字几"

## 原理解析

### 从图片到数字

一张 28×28 的图片有 784 个像素。网络需要 784 个输入，每个像素对应一个输入。

```
图片 (28×28) → 展平 → 784 个数字的一维数组 → 网络的 784 个输入
```

**为什么要展平？**

因为我们的网络输入层是一维的（784 个节点）。每个节点接收一个像素值。

### 网络输出是什么？

网络的输出层有 10 个节点，分别对应数字 0-9。

输出类似这样：
```
[0.01, 0.02, 0.85, 0.03, 0.01, 0.02, 0.01, 0.03, 0.01, 0.01]
         ↑
       索引 2 → 数字 2 的概率是 85%
```

**最大概率对应的索引就是答案。**

### 批处理：一次处理多张图片

一次处理一张图片太慢了。批处理就是：**把多张图片打包成一个矩阵，一次性计算**。

```
一张图片:    (784,)       ← 1 行 784 列
100 张图片: (100, 784)    ← 100 行 784 列
```

**类比：**
- 一次处理一张 = 一个一个收银
- 批处理 = 开 100 个收银台同时收银

**批处理的好处：**
- 快（利用 CPU/GPU 并行计算）
- 代码更简洁（不需要 for 循环一张一张处理）

## 代码实战

### 完整可运行代码（模拟推理）

由于实际 MNIST 数据需要下载，我们用随机数据模拟推理过程：

```python
import numpy as np

# === 1. 定义激活函数 ===
def sigmoid(x):
    """Sigmoid 函数"""
    # 数值稳定版：防止溢出
    return np.where(x >= 0, 
                    1 / (1 + np.exp(-x)), 
                    np.exp(x) / (1 + np.exp(x)))

def softmax(a):
    """Softmax 函数：把输出变成概率"""
    c = np.max(a)
    exp_a = np.exp(a - c)
    return exp_a / np.sum(exp_a)

# === 2. 初始化网络参数（模拟训练好的权重）===
np.random.seed(42)  # 固定随机种子，每次运行结果一样

network = {}
network['W1'] = np.random.randn(784, 50) * 0.01   # 784→50
network['b1'] = np.zeros(50)
network['W2'] = np.random.randn(50, 100) * 0.01   # 50→100
network['b2'] = np.zeros(100)
network['W3'] = np.random.randn(100, 10) * 0.01   # 100→10
network['b3'] = np.zeros(10)

# === 3. 定义前向传播函数 ===
def forward(network, x):
    """从输入到输出的完整计算"""
    W1 = network['W1']
    W2 = network['W2']
    W3 = network['W3']
    b1 = network['b1']
    b2 = network['b2']
    b3 = network['b3']
    
    # 第 1 层：784 → 50
    a1 = np.dot(x, W1) + b1
    z1 = sigmoid(a1)
    
    # 第 2 层：50 → 100
    a2 = np.dot(z1, W2) + b2
    z2 = sigmoid(a2)
    
    # 第 3 层：100 → 10
    a3 = np.dot(z2, W3) + b3
    y = softmax(a3)  # 输出层用 Softmax，变成概率
    
    return y

# === 4. 模拟输入：10 张"图片"（随机数据）===
# 每张"图片"是 784 个随机数（模拟 28×28 像素）
x_batch = np.random.randn(10, 784)

print("输入形状:", x_batch.shape)  # (10, 784)

# === 5. 推理 ===
y = forward(network, x_batch)

print("输出形状:", y.shape)        # (10, 10)
print()

# === 6. 解读结果 ===
for i in range(10):
    predicted = np.argmax(y[i])    # 概率最大的索引 = 预测的数字
    confidence = y[i][predicted]   # 最大概率
    print(f"图片 {i}: 预测数字 = {predicted}, 置信度 = {confidence:.2%}")

print()
print("输出总和（验证 Softmax）:", y[0].sum())
```

### 运行结果

```
输入形状: (10, 784)
输出形状: (10, 10)

图片 0: 预测数字 = 7, 置信度 = 12.34%
图片 1: 预测数字 = 3, 置信度 = 11.89%
图片 2: 预测数字 = 5, 置信度 = 10.45%
...
输出总和（验证 Softmax）: 1.0
```

**解读：**
- 因为权重是随机的（没经过训练），所以准确率很低，每个数字的概率差不多。
- 但流程是对的：输入 784 个数字 → 网络处理 → 输出 10 个概率 → 选最大的。
- 训练之后，正确答案的概率会接近 100%，其他接近 0%。

### 关键步骤拆解

**第 1 步：准备输入**
```python
x_batch = np.random.randn(10, 784)
```
- 10 行 = 10 张图片
- 784 列 = 每张图 784 个像素

**第 2 步：前向传播**
```python
y = forward(network, x_batch)
```
- 输入 `(10, 784)` → 第 1 层 `(10, 50)` → 第 2 层 `(10, 100)` → 第 3 层 `(10, 10)`
- 每一行是一张图片的输出

**第 3 步：找最大值**
```python
predicted = np.argmax(y[i])
```
- `argmax` 返回最大值的索引
- 索引 0 = 数字 0，索引 1 = 数字 1，……

## 避坑指南

1. **权重是随机的，所以准确率低** - 这个代码只是演示推理流程。实际应该加载训练好的权重。第 4 章会学怎么训练。

2. **`argmax` 的 axis 参数** - `np.argmax(y[i])` 对一行找最大值。如果是批处理，用 `np.argmax(y, axis=1)` 对每一行找最大值。

3. **Softmax 输出总和一定是 1** - 这是验证代码是否正确的一个好方法。
