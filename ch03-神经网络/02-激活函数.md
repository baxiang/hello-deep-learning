# 3.2 激活函数

## 学习目标

- 理解为什么激活函数必须是"非线性"的
- 掌握 Sigmoid、ReLU、Softmax 三种函数
- 能用代码实现并运行这三种函数

## 开篇问题

**上一章我们学了神经网络的计算过程：加权和 → 激活函数 → 传给下一层。**

有个关键问题：**为什么必须有激活函数？**

如果没有激活函数，会发生什么？

先看答案：如果激活函数是线性的，多层网络会退化成一层。也就是说，加了再多层也没用。

本章会用具体的数值证明这一点。

## 核心概念

### 数学本质：激活函数的作用

**激活函数的核心作用：引入非线性。**

为什么需要非线性？

看看这个例子：

```
假设有两个神经元，都只用"线性"激活函数（y = 2x）

神经元1：输入 3 → 加权和 5 → 激活 2×5 = 10
神经元2：输入 10 → 加权和 8 → 激活 2×8 = 16

整个流程：3 → 10 → 16

这等价于：直接从 3 到 16，也就是 3 × 某个系数 = 16
```

**数值证明：线性激活让多层无效**

假设一个 3 层网络，每层的权重和线性激活：
- 第 1 层权重：w1 = 0.5，线性激活 y = 2x
- 第 2 层权重：w2 = 0.3，线性激活 y = 2x
- 第 3 层权重：w3 = 0.4，线性激活 y = 2x

输入 x = 10：

```
第1层：10 × 0.5 × 2 = 10
第2层：10 × 0.3 × 2 = 6
第3层：6 × 0.4 × 2 = 4.8

最终输出：4.8

简化成一层：10 × 0.5 × 0.3 × 0.4 × 2 × 2 × 2 = 10 × 0.48 = 4.8
```

你看，三层网络 = 一层网络。所以线性激活让多层失去意义。

**结论：必须用非线性激活函数。**

### 什么是激活函数？

激活函数是神经网络中的"转换器"。

**类比：** 想象你有一个音量旋钮。输入信号是原始声音大小，激活函数决定"实际播放多大的声音"。

- 有的旋钮会把超大声音压低（Sigmoid）
- 有的旋钮会把负音量静音（ReLU）
- 有的旋钮会把多个声音调成比例（Softmax）

### 为什么必须是"非线性"的？

这是理解神经网络最关键的一点。

**线性函数是什么？** 就是"输入乘以一个固定倍数"。比如 `y = 2x`，输入翻倍，输出也翻倍。

**如果激活函数是线性的，会发生什么？**

假设网络有两层，每层的激活函数都是 `y = 2x`：
- 第 1 层：输入 x → 输出 2x
- 第 2 层：输入 2x → 输出 2×(2x) = 4x

你看，两层合起来还是线性的（`y = 4x`）。那加多少层都没区别，一层就够了。

**这到底意味着什么？**

**类比：** 你用两个放大镜看东西。如果两个放大镜的效果只是"放大 2 倍"，那用两个和一个没区别（都是放大 4 倍）。但如果第一个放大镜会变色，第二个会扭曲，那效果就完全不一样了。激活函数的"非线性"就是那种"变色+扭曲"的效果，让多层网络能做单层做不到的事情。

## 原理解析

### 1. Sigmoid 函数

**公式（不用背，理解意思就行）：**

```
输出 = 1 / (1 + e 的负 x 次方)
```

**它做了什么：**

不管输入多大或多小，输出永远在 0 到 1 之间。

**生活中的类比：**

Sigmoid 像一个"情绪稳定器"。不管遇到多大的好事或多坏的坏事，它都让你保持冷静，反应不会太极端。

- 输入 100 → 输出 0.999...（接近 1，但不会超过 1）
- 输入 -100 → 输出 0.000...1（接近 0，但不会小于 0）
- 输入 0 → 输出 0.5

**为什么早期用它？**

因为输出在 0~1 之间，很像"概率"。比如输出 0.8 可以理解为"80% 的把握"。

**为什么不常用了？**

当输入很大或很小时，曲线变得很平。这意味着"梯度"（后面会学）几乎为 0，网络学不动了。

**类比：** 一个老师，学生考 90 分和 100 分他都给"很好"的评价，学生考 5 分和 0 分他都给"很差"的评价。那学生就不知道该怎么改进了。

### 2. ReLU 函数

**公式（超简单）：**

```
如果输入 > 0，输出 = 输入
如果输入 ≤ 0，输出 = 0
```

**它做了什么：**

正数直接放过，负数变成 0。

**数值演示：ReLU 如何"过滤"信息**

```
输入神经元值：
  神经元1：-26.6（负数）
  神经元2：-47.1（负数）
  神经元3：12.3（正数）

ReLU 处理：
  神经元1：max(0, -26.6) = 0
  神经元2：max(0, -47.1) = 0
  神经元3：max(0, 12.3) = 12.3

输出：
  [0, 0, 12.3]
```

**关键观察：**
- 负数全部变成 0 → 被"过滤掉"
- 正数保持原值 → 被"保留"
- ReLU 在做信息筛选

这就像一个"过滤器"，只保留正信号，负信号被拦截。

**生活中的类比：**

ReLU 像一个"单行道"：
- 正向行驶（正数）→ 直接通过
- 逆向行驶（负数）→ 拦住，不许过

**为什么现在最常用？**

- 计算超快（就一个判断）
- 不会"学不动"（正数部分梯度永远是 1）

### 3. Softmax 函数："概率"的起源

**开篇问题：神经网络的输出如何变成"概率"？**

**困境：输出值与理想值的对比问题**

```
例子：识别叶子 vs 花朵

神经网络输出：
  预测1：（叶子=5，花朵=1）→ 选择叶子 ✓ 正确
  预测2：（叶子=0，花朵=1）→ 选择花朵 ✗ 错误

期望的理想值：
  叶子正确时：理想值=（1，0）
  
计算损失（与理想值差距）：
  预测1：（5，1）与（1，0）差距大 → 损失大
  预测2：（0，1）与（1，0）差距小 → 损失小

问题：
  预测正确 → 损失大？
  预测错误 → 损失小？
  
矛盾：预测正确的反而损失大！
```

**解决思路：把输出变成概率**

```
需要满足三个条件：
  1. 所有值在0-1之间（像概率）
  2. 总和等于1（概率总和）
  3. 保持大小关系（5>1 → 概率5>概率1）

例子：
  输出（5，1）→ 概率（0.98，0.02）
  输出（0，1）→ 概率（0.27，0.73）
  
现在：
  预测1正确：概率（0.98，0.02）接近理想（1，0）→ 损失小 ✓
  鐰测2错误：概率（0.27，0.73）远离理想（1，0）→ 损失大 ✓
  
结果：正确预测损失小，错误预测损失大 → 符合逻辑
```

**公式（不用背，理解意思）：**

```
输出_k = e 的 a_k 次方 / 所有 e 的 a_i 次方之和
```

**它做了什么：**

把一组数字变成比例，所有比例加起来等于 1。

**数值演示：Softmax如何计算**

```
输入分数：（2，1，0.1）

步骤1：指数运算（为什么用e？）

  e² ≈ 7.389
  e¹ ≈ 2.718
  e⁰·¹ ≈ 1.105

为什么用指数函数？
  ① 差异放大：放大决策信号（差距2和1 → 指数差距7.389和2.718）
  ② 正数保障：输入负数，结果仍为正
  ③ 单调性：保持大小顺序（5>1 → e^5 > e^1）

步骤2：归一化（除以总和）

  总和 = 7.389 + 2.718 + 1.105 ≈ 11.212
  
  概率计算：
    分数2：7.389 / 11.212 ≈ 0.659（65.9%）
    分数1：2.718 / 11.212 ≈ 0.242（24.2%）
    分数0.1：1.105 / 11.212 ≈ 0.099（9.9%）
  
验证：
  概率都在0-1之间 ✓
  总和 = 0.659 + 0.242 + 0.099 = 1.0 ✓
  大小关系保持：2>1>0.1 → 0.659>0.242>0.099 ✓
```

**生活中的类比：**

Softmax 像一个"分蛋糕的人"。你给它几个数字，它把这些数字变成每个人分到蛋糕的比例。

比如输入 `[1, 2, 3]`：
- 原始总和 = 6
- Softmax 后 ≈ `[0.09, 0.24, 0.67]`
- 加起来 = 1.0（一整个蛋糕）

**关键理解：** 最大的数字会分到最大的比例，但不会独占。3 比 2 大一点，所以 0.67 比 0.24 大不少（指数放大了差距）。

### Softmax的价值：不只是输出概率

**应用1：输出层预测**

```
识别手写数字（0-9，共10个输出）

神经网络输出原始分数：
  （3，-1，5，0，...）
  
Softmax转换：
  （0.02，0.001，0.85，0.01，...）
  
理解：
  数字"2"的概率85% → 最可能是数字2
```

**应用2：注意力机制权重**

```
Transformer中的注意力计算

Query与Key的匹配分数：
  （2.5，0.3，1.8）
  
Softmax转换注意力权重：
  （0.70，0.08，0.22）
  
理解：
  第1个词关注度70%
  第2个词关注度8%
  第3个词关注度22%
```

**应用3：生成多样性**

```
语言模型生成："Humpty Du___"

原始分数：
  'm'：0.45
  'u'：0.50（最高）
  'a'：0.05
  
直接选最高：选'u' → "Humpty Duu..." 错误路径
  
Softmax概率分布：
  'm'：45%概率
  'u'：50%概率
  'a'：5%概率
  
按概率采样：
  有45%概率选'm' → "Humpty Dum..." 正确路径
  
价值：
  Softmax赋予次优选项概率
  模型可以探索多种可能
  在犯错时有机会修正
```

### 温度参数：控制生成特性

**公式：**

```
输出_k = e 的 (a_k / T) 次方 / 所有 e 的 (a_i / T) 次方之和

T = 温度参数
```

**数值演示：温度如何影响概率**

```
原始分数：（5，1）

=== 低温度（T=0.5）===

指数运算：
  e^(5/0.5) = e^10 ≈ 22026
  e^(1/0.5) = e^2 ≈ 7.389
  
概率：
  分数5：22026/22033 ≈ 99.97%
  分数1：7/22033 ≈ 0.03%

效果：概率分布尖锐 → 几乎只选最高分 → 确定性强

=== 高温度（T=2）===

指数运算：
  e^(5/2) = e^2.5 ≈ 12.18
  e^(1/2) = e^0.5 ≈ 1.65
  
概率：
  分数5：12.18/13.83 ≈ 88%
  分数1：1.65/13.83 ≈ 12%

效果：概率分布平缓 → 次优选项也有机会 → 创造性强
```

**类比：餐厅选择**

```
两家餐厅评分：5分和4.7分

低温度（T小）：
  差距放大 → 只选5分的餐厅 → 保守

高温度（T大）：
  差距缩小 → 两家轮流尝试 → 探索
```

**实际应用：**

|温度 | 效果 | 适用场景 |
|------|------|----------|
| T<1（低温） | 确定性强、保守 | 代码生成、准确回答 |
| T=1（默认） | 平衡 | 普通对话 |
| T>1（高温） | 创造性强、多样 | 写作、创意生成 |

**用在哪里？**

只在输出层用，用于分类任务。比如识别手写数字，10 个输出就是 10 个数字的概率。

## 代码实战

### 完整可运行代码

```python
import numpy as np

# === 1. Sigmoid 函数 ===
def sigmoid(x):
    """把任意数字压缩到 0~1 之间"""
    return 1 / (1 + np.exp(-x))

# === 2. ReLU 函数 ===
def relu(x):
    """正数直接输出，负数变成 0"""
    return np.maximum(0, x)

# === 3. Softmax 函数（数值稳定版）===
def softmax(a):
    """把一组数字变成比例，总和为 1"""
    # 减去最大值防止溢出（后面解释为什么）
    c = np.max(a)
    exp_a = np.exp(a - c)
    sum_exp_a = np.sum(exp_a)
    return exp_a / sum_exp_a

# === 测试一下 ===
# 准备测试数据：有正有负有零
x = np.array([-2.0, -1.0, 0.0, 1.0, 2.0])

print("输入:", x)
print()

# 测试 Sigmoid
print("Sigmoid 输出:", sigmoid(x))
print("  → 所有值都在 0~1 之间")
print()

# 测试 ReLU
print("ReLU 输出:", relu(x))
print("  → 负数变 0，正数不变")
print()

# 测试 Softmax
print("Softmax 输出:", softmax(x))
print("  → 所有值加起来 =", np.sum(softmax(x)))
```

### 运行结果

```
输入: [-2. -1.  0.  1.  2.]

Sigmoid 输出: [0.11920292 0.26894142 0.5        0.73105858 0.88079708]
  → 所有值都在 0~1 之间

ReLU 输出: [0. 0. 0. 1. 2.]
  → 负数变 0，正数不变

Softmax 输出: [0.01587624 0.04317157 0.11750144 0.3194532  0.86799755]
  → 所有值加起来 = 1.0
```

### 逐行解读

**Sigmoid 部分：**
```python
return 1 / (1 + np.exp(-x))
```
- `np.exp(-x)` 计算 e 的负 x 次方。
- `1 / (1 + ...)` 把结果压缩到 0~1 之间。

**ReLU 部分：**
```python
return np.maximum(0, x)
```
- `np.maximum` 逐个比较两个数组，取大的那个。
- 和 0 比较，负数就被 0 替代了。

**Softmax 部分：**
```python
c = np.max(a)              # 找最大值
exp_a = np.exp(a - c)      # 每个数减去最大值后求 e 的次方
sum_exp_a = np.sum(exp_a)  # 求和
return exp_a / sum_exp_a   # 每个数除以总和
```
- 为什么要减去最大值？因为 `e^1000` 会大到计算机存不下（溢出）。减去最大值后，最大的变成 `e^0 = 1`，就不会溢出了。

## 避坑指南

1. **Softmax 不减最大值会溢出** - 输入 `[1000, 1000, 1000]` 时，`e^1000` 会溢出。一定要先减最大值。

2. **Sigmoid 在深层网络中会导致"梯度消失"** - 输入很大或很小时，曲线太平，网络学不动。深层网络优先用 ReLU。

3. **激活函数选哪个？**
   - 隐藏层：优先 ReLU
   - 输出层：分类用 Softmax，回归用恒等函数（输入什么输出什么）

4. **Softmax不是真正的"概率"** - 它是归一化的分数，只是看起来像概率。模型输出的"概率"90%不代表真实90%准确率。

5. **温度参数的陷阱**
   - T太小（如0.1）：几乎只选最高分 → 失去多样性，可能陷入重复
   - T太大（如3）：概率过于平均 → 生成质量下降，可能胡言乱语
   - 建议：代码生成T=0.2-0.5，创意写作T=0.7-1.2

6. **Softmax的梯度消失问题**
   - 当某个分数远大于其他分数时（如100 vs 1），其他概率接近0
   - 这些接近0的神经元梯度也接近0 → 学不动
   - 解决：初始化权重不宜过大，使用温度参数调节

## 总结

**激活函数的本质：引入非线性。**

```
线性激活 → 多层等于一层 → 失去意义
非线性激活 → 每层都能学到新特征 → 深层网络有意义
```

**三种激活函数对比：**

|函数 | 作用 | 输出范围 | 用途 |
|------|------|----------|------|
| Sigmoid | 压缩到0-1 | (0, 1) | 早期网络，输出层 |
| ReLU | 负数变0 | [0, +∞) | 隐藏层（主流） |
| Softmax | 变概率 | (0, 1)，总和=1 | 输出层（分类） |

**Softmax的核心价值：**

```
神经网络输出 → Softmax → 概率分布

三个作用：
  ① 输出层预测：分类任务的置信度
  ② 注意力权重：分配关注度
  ③ 生成多样性：探索多种可能

温度参数：
  T小 → 确定性强 → 代码生成
  T大 → 创造性强 → 创意写作
```

**记住：**
- 没有激活函数 → 深层网络无意义
- 隐藏层优先ReLU（快、不梯度消失）
- 输出层分类用Softmax（变成概率）
- 温度参数控制生成特性
