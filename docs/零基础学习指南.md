### 学习重点

```python
import numpy as np

A.reshape(4, 1)    # 改变形状
```

**感知机概念**

理解核心思想：**感知机 = 简单的判断器**

```
输入 → 加权求和 → 判断 → 输出

例子：判断是否买手机
  价格权重 = -0.5（价格高不想买）
  品牌权重 = 0.3（喜欢这个品牌）

  总分 = 价格 × (-0.5) + 品牌喜爱度 × 0.3
  如果总分 > 阈值 → 买
  否则 → 不买
```

### 学习重点

**神经网络结构**

理解核心：**神经网络 = 很多感知机连在一起**

```
输入层 → 隐藏层 → 输出层

每一层：
  输入 → 矩阵乘法 → 激活函数 → 输出

激活函数的作用：
  - sigmoid：把输出压缩到 0~1之间
  - ReLU：负数变0，正数不变
```

**学习过程**

理解核心：**学习 = 找最优参数**

```
1. 初始化：随机参数
2. 前向传播：计算预测结果
3. 计算损失：预测和真实差距多大
4. 反向传播：计算梯度（用导数）
5. 更新参数：往梯度反方向走一步
6. 重复2-5，直到损失很小
```

**梯度下降直觉**

```
想象蒙眼下山：
- 感觉脚下哪边最陡（计算梯度）
- 往下坡方向走一步（更新参数）
- 重复，直到到平地（最优解）

学习率 = 每步走多远
  - 太大：可能跨过山谷
  - 太小：走太慢
```

### 数学知识补充

**导数（高中已学，复习一下）**

```
函数 y = x²
导数 dy/dx = 2x

意义：x=3 时，y=9；x稍微增加，y会增加多少？
答案：增加 2×3 = 6倍（这就是导数）

深度学习中：
  损失函数 L(权重)
  导数 = 权重改变时，损失改变多少
  梯度 = 各参数导数的组合
```

**矩阵乘法（新知识，用类比理解）**

```
矩阵 = 数字表格

A = [[1, 2],     B = [[5, 6],
     [3, 4]]          [7, 8]]

A × B 的计算：
  第1行第1列 = 1×5 + 2×7 = 19
  第1行第2列 = 1×6 + 2×8 = 22
  ...

类比：矩阵乘法 = 批量计算
  一次处理多个数据，而不是逐个处理
```

### 学习重点

**反向传播**

理解核心：**反向传播 = 高效计算梯度**

```
计算图：把计算过程画成图

正向：输入 → 计算 → 输出
反向：输出误差 → 反向传递 → 各参数的梯度

链式法则（高中数学）：
  y = f(g(x))
  dy/dx = dy/dg ×dg/x

  意思：总导数 = 各环节导数的乘积
```

**优化方法对比**

```
SGD（随机梯度下降）：
  权重 = 权重 - 学习率 × 梯度
  缺点：走冤枉路，震荡

Momentum（动量）：
  加入惯性，方向一致时加速
  比SGD更快收敛

Adam：
  结合动量 + 自适应学习率
  目前最常用，推荐默认使用
```

## 常见问题解答

### Q：我没有线性代数基础，能学吗？

**可以。** 项目中会用通俗语言解释矩阵运算：

```
向量 = 一排数字 [1, 2, 3]
矩阵 = 数字表格 [[1,2], [3,4]]
矩阵乘法 = 批量计算，一次处理多个数据
```

你只需要理解"批量计算"的概念，具体运算交给NumPy。

### Q：高中导数够用吗？

**够用。** 你只需要理解：

```
导数 = 函数的变化率
  - x增加一点，y增加多少？
  - 这就是导数告诉你的

梯度下降：
  - 梯度 = 导数的组合
  - 沿梯度反方向走 = 往"下降"方向走
```

不需要多元微积分的严格证明。

### Q：为什么用纯NumPy，不用PyTorch？

**原因：**

1. **理解原理**：框架封装了细节，纯NumPy能看到原始实现
2. **教学友好**：每一行代码都清晰可见
3. **降低门槛**：不需要学习框架API

学会原理后，再用PyTorch会更容易。

### Q：学完能做什么？

**学完后你能：**

1. 理解深度学习的核心原理
2. 理解常见模型（CNN、RNN、Transformer）
3. 能够阅读更进阶的教程（如《动手学深度学习》）
4. 使用PyTorch等框架时知道"底层发生了什么"

### Q：遇到不懂的地方怎么办？

**建议：**

1. 先跳过，继续往后看
2. 后面的内容可能帮你理解前面
3. 跑代码实验，建立直觉
4. 不纠结细节，理解大意即可
