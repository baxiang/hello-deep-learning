## 第四层：多头注意力展开

```
┌─────────────────────────────────────────────────────────────┐
│                    多头注意力机制                             │
│                                                               │
│输入向量：[v1, v2, v3]                                        │
│    ↓                                                          │
│拆分成多个头（假设3头）：                                      │
│    ↓                                                          │
│  ┌───────┐┌───────┐┌───────┐                              │
│  │ 头1   ││ 头2   ││ 头3   │                              │
│  │关注语法││关注语义││关注情感│                              │
│  └───────┘└───────┘└───────┘                              │
│    ↓         ↓         ↓                                     │
│头1结果  头2结果  头3结果                                      │
│    ↓         ↓         ↓                                     │
│拼接 →线性变换 →输出                                          │
│                                                               │
│数值演示（每个头的计算）：                                      │
│                                                               │
│  Query（查询）："我想找什么？"                                 │
│    Key（键）： "我有什么信息？"                                 │
│    Value（值）： "我的内容是什么？"                             │
│                                                               │
│  注意力分数 = Query × Key                                     │
│  概率 = Softmax(分数)                                          │
│  输出 = 概率 × Value                                           │
│                                                               │
│  示例（头1）：                                                 │
│    Query(词1) = [0.5, 0.3]                                    │
│    Key(词2) = [0.4, 0.2]                                      │
│    分数 = 0.5×0.4 + 0.3×0.2 = 0.26                            │
│    概率 = softmax([0.26, ...])                                │
│    Value(词2) = [0.6, 0.1]                                    │
│    输出 = 概率 × Value                                         │
└─────────────────────────────────────────────────────────────┘
```

## 第八层：残差连接的作用

```
┌─────────────────────────────────────────────────────────────┐
│                    残差连接的作用                             │
│                                                               │
│┌─────────────────────────────────────────────────────┐    │
││ 传统做法（无残差）：                                 │    │
││                                                       │    │
││ 输入：x = [1.0, 2.0]                                 │    │
││   ↓                                                   │    │
││ 处理层1：F1(x) = [0.5, -1.0]                         │    │
││   ↓                                                   │    │
││ 处理层2：F2(F1) = [-2.0, 0.3]                        │    │
││   ↓                                                   │    │
││ 处理层3：F3(F2) = [0.1, -0.5]                        │    │
││                                                       │    │
││ 问题：                                                │    │
││   层数多了，原始信息 x 被淹没                        │    │
││   梯度传回时，每层乘系数 →可能消失                   │    │
│└─────────────────────────────────────────────────────┘    │
│                                                               │
│┌─────────────────────────────────────────────────────┐    │
││ 残差连接：                                           │    │
││                                                       │    │
││ 输入：x = [1.0, 2.0]                                 │    │
││   ↓                                                   │    │
││ x + F1(x) = [1.5, 1.0]  ← 保留了 x                   │    │
││   ↓                                                   │    │
││ (x+F1) + F2(x+F1) = [?, ?] ← 又保留了前一层         │    │
││   ↓                                                   │    │
││ ...                                                   │    │
││                                                       │    │
││ 优点：                                                │    │
││   即使 F 很差，输出仍有原始成分                      │    │
││   梯度通过捷径传回，不消失                            │    │
││   深层网络（100+层）可训练                            │    │
││                                                       │    │
││ 公式：输出 = F(x) + x                                 │    │
│└─────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘
```

## 关键组件总结表

|组件 |位置 |作用 |数值范围 |
|------|------|------|----------|
| 词嵌入 |输入层 | token →向量 | -0.1 ~ 0.1 |
| 位置嵌入 |输入层 | 添加位置信息 | -1 ~ 1 |
| 多头注意力 | Block | 理解上下文 | - |
| 残差连接 | Block | 保留原始信息 | - |
| 层归一化 | Block | 稳定数值 | 标准化后 |
| 前馈网络 | Block | 深度加工 | - |
| ReLU | 前馈网络 | 负数→0，正数保留 | 0 ~ 正数 |
| 线性层 | 输出层 | 向量→词表评分 | - |
| Softmax | 输出层 | 评分→概率 | 0 ~ 1 |

## 核心公式总结

```
1. 词嵌入 + 位置嵌入：
   输入 = E(token) + P(position)

2. GPT Block：
   输出 = x + 注意力(归一化(x))
   输出 = y + 前馈网络(归一化(y))

3. 多头注意力：
   注意力分数 = Query × Key
   概率 = Softmax(分数)
   输出 = 概率 × Value

4. 残差连接：
   输出 = F(x) + x

5. 层归一化：
   归一化后 = (值 - 均值) / 标准差

6. Softmax：
   概率_k = exp(评分_k) / Σ exp(评分_i)

7. 自回归生成：
   下一个词 = argmax(概率)
   新输入 = 旧输入 + 下一个词
```

## 记忆要点

**LLM 的本质：**

```
输入 → GPT → 概率 → 下一个词
拼接 → 再输入 → GPT → 概率 → 下一个词
...（重复）
```

**三大核心：**
