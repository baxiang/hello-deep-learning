# A.1 神经网络：一个处理数字的黑盒

## 学习目标

- 理解神经网络的数学本质
- 用具体数值演示计算过程
- 明白"一切都是数字"的核心思想

## 开篇问题

**神经网络看起来很神奇：给它图片，它能识别；给它文字，它能生成回复。**

问题是：**神经网络到底在做什么？**

答案：只在做加法、乘法，处理数字。

## 核心概念

### 数学本质：一切都是数字

**关键理解：神经网络不理解任何含义，只处理数字。**

举例：

| 任务 | 输入（数字） | 输出（数字） | 你赋予的含义 |
|------|------------|------------|------------|
| 图片分类 | 像素值（0-255） | 分类概率 | "这是猫的概率" |
| 情感分析 | 字符编号 | 正负概率 | "这是正面评论" |
| 语言生成 | 字符编号 | 字符概率 | "下一个字是'的'" |
| 天气预测 | 温度、湿度 | 概率 | "下雨概率" |

**对神经网络来说：**
- 不理解"猫"、"正面评论"、"下雨"的含义
- 只理解数字的输入和输出
- 你赋予数字含义，模型只负责映射

### 一个具体的神经网络例子

假设我们要识别"叶子"还是"花朵"。

**输入：** 4 个数字（RGB 颜色 + 体积）
```
叶子样本：
  R = 32（红色通道）
  G = 107（绿色通道）
  B = 56（蓝色通道）
  Vol = 11.2（体积，毫升）
```

**输出：** 2 个数字（叶子概率 + 花朵概率）
```
期望输出：
  叶子概率 = 1（100%）
  花朵概率 = 0（0%）
```

**网络结构：**
```
输入层（4个神经元）
  ↓
隐藏层（3个神经元）
  ↓
输出层（2个神经元）
```

## 数值演示：完整的计算过程

### 第 1 步：加权和计算

隐藏层第 1 个神经元：

```
输入值：R=32, G=107, B=56, Vol=11.2
权重：w1=0.1, w2=-0.29, w3=-0.07, w4=0.46
偏置：b=0.25

计算：
  加权和 = 32×0.1 + 107×(-0.29) + 56×(-0.07) + 11.2×0.46 + 0.25
  
  = 3.2 + (-31.03) + (-3.92) + 5.152 + 0.25
  
  = -26.35
```

隐藏层第 2 个神经元：

```
权重不同，计算类似：
  加权和 = 32×w1' + 107×w2' + 56×w3' + 11.2×w4' + b'
  
  假设结果是：-47.1
```

隐藏层第 3 个神经元：

```
计算结果假设是：12.3
```

### 第 2 步：激活函数处理

使用 ReLU（负数变成 0，正数保持）：

```
神经元1：输入 -26.35（负数）
  ReLU：max(0, -26.35) = 0
  
神经元2：输入 -47.1（负数）
  ReLU：max(0, -47.1) = 0
  
神经元3：输入 12.3（正数）
  ReLU：max(0, 12.3) = 12.3

激活后输出：[0, 0, 12.3]
```

**关键观察：**
- 负的加权和被"过滤掉" → 变成 0
- 正的加权和被"保留" → 保持原值
- ReLU 在筛选信息

### 第 3 步：输出层计算

```
隐藏层输出：[0, 0, 12.3]

输出层加权和：
  叶子神经元 = 0×w1 + 0×w2 + 12.3×w3 + b_叶子
  花朵神经元 = 0×w1' + 0×w2' + 12.3×w3' + b_花朵

假设结果是：
  叶子神经元：0.8
  花朵神经元：0.2

最终输出：[0.8, 0.2]
```

### 第 4 步：解释输出

```
输出数字：[0.8, 0.2]

你的解释：
  叶子概率 = 0.8 → 80% 是叶子
  花朵概率 = 0.2 → 20% 是花朵

结论：这是叶子（因为 0.8 > 0.2）
```

## 为什么叫"神经"网络？

**灵感来自生物大脑。**

| 生物神经元 | 人工神经元 |
|-----------|-----------|
| 接收多个信号 | 接收多个数字 |
| 信号总和超过阈值 | 加权和超过某个值 |
| "放电"输出信号 | 输出一个数字 |

本质相同，只是用数字代替电信号。

## 关键理解总结

1. **神经网络不理解含义**
   - 不理解"叶子"、"花朵"、"RGB"
   - 只理解输入数字和输出数字
   
2. **你赋予数字含义**
   - 你说"输入是 RGB" → 模型处理 RGB
   - 你说"输出是概率" → 模型输出概率
   
3. **计算过程很简单**
   - 加权和：加法和乘法
   - 激活函数：简单的判断（正负）
   - 就这么简单，没有神秘

4. **同一个网络可以做不同任务**
   - 输入天气数据 → 输出下雨概率
   - 输入股价数据 → 输出涨跌概率
   - 网络只处理数字，任务由你定义

## 避坑指南

1. **不要被图表迷惑** - 看起来复杂的网络图，背后只是加法乘法
2. **不要过度理解含义** - 模型不理解"叶子"，只处理数字
3. **记住数学本质** - 输入数字 → 加权求和 → 激活 → 输出数字

## 课后思考

1. 如果把所有权重设为 0，输出会是什么？
2. 为什么需要激活函数？（下一节会详细讲）
3. 同一个网络能否同时识别图片和预测天气？（答案：能，只要权重对）

# A.2 训练：如何让网络找到好权重

## 学习目标

- 理解训练的本质：调整权重
- 用数值演示误差如何缩小
- 明白"学习"就是不断试错

## 开篇问题

**上一节我们假设"权重已经训练好"。**

问题是：**这些权重怎么来的？**

两种可能：
1. 人手动设 → 不现实（参数太多）
2. 网络自己找 → 这就是"训练"

## 核心概念

### 训练的本质：找到合适的权重

**目标：找到一组权重，让网络输出符合期望。**

数值演示：

```
期望输出：叶子=1，花朵=0（100%是叶子）

当前权重（随机）：输出=叶子0.6，花朵0.4

差距：
  叶子：期望1，实际0.6 → 差0.4
  花朵：期望0，实际0.4 → 差0.4

总误差 = 0.4 + 0.4 = 0.8（很大！）

训练目标：调整权重，让误差变小
```

### 训练需要什么？

**三个要素：**

1. **训练数据** - 告诉网络什么是对的
   ```
   已标注的叶子图片：标注"叶子"
   已标注的花朵图片：标注"花朵"
   ```

2. **损失函数** - 告诉网络错在哪里
   ```
   误差 = |期望 - 实际|
   叶子误差 = |1 - 0.6| = 0.4
   花朵误差 = |0 - 0.4| = 0.4
   ```

3. **调整方法** - 告诉网络怎么改进
   ```
   叶子输出小 → 增加叶子权重
   花朵输出大 → 减少花朵权重
   ```

## 数值演示：一个训练回合

### 第 1 步：随机初始化权重

```
初始权重（随机）：
  输入→隐藏层权重：[0.1, -0.29, -0.07, 0.46]
  隐藏→输出层权重：[0.2, 0.1]
```

### 第 2 步：输入数据，计算输出

```
输入：叶子样本（R=32, G=107, B=56, Vol=11.2）

前向传播：
  加权和 → 激活 → 输出
  
输出：叶子=0.6，花朵=0.4
```

### 第 3 步：计算误差

```
期望输出：叶子=1，花朵=0
实际输出：叶子=0.6，花朵=0.4

误差：
  叶子：|1 - 0.6| = 0.4
  花朵：|0 - 0.4| = 0.4
  
总误差 = 0.8
```

### 第 4 步：分析改进方向

```
问题诊断：
  叶子输出0.6 → 太小（期望1）
  花朵输出0.4 → 太大（期望0）

改进方向：
  增加"叶子相关"权重 → 叶子输出变大
  减少"花朵相关"权重 → 花朵输出变小
```

### 第 5 步：调整权重

```
调整幅度：学习率=0.01（每次调整很小）

叶子权重调整：
  原值：0.2
  增加：0.2 + 0.01×(误差) = 0.2 + 0.004 = 0.204

花朵权重调整：
  原值：0.1
  减少：0.1 - 0.01×(误差) = 0.1 - 0.004 = 0.096
```

### 第 6 步：验证改进

```
用新权重再次计算：
  输出：叶子=0.65，花朵=0.35
  
新误差：
  叶子：|1 - 0.65| = 0.35
  花朵：|0 - 0.35| = 0.35
  
总误差 = 0.7（比0.8小了！）
```

**关键观察：**
- 误差从 0.8 → 0.7，变小了
- 再重复多次，误差会越来越小
- 最终权重能准确识别叶子

## 训练过程的本质

**类比：学骑自行车**

| 训练步骤 | 学骑车类比 |
|---------|-----------|
| 随机初始 | 一开始不会骑，歪歪扭扭 |
| 计算输出 | 实际骑车的表现 |
| 计算误差 | 看是否摔倒、偏离路线 |
| 分析改进 | 思考哪里不对、怎么调整 |
| 调整权重 | 调整身体姿势、力度 |
| 重复多次 | 摔了几百次后，学会了 |

**关键：不是"背答案"，而是"不断试错调整"。**

## 为什么需要大量训练？

**一次调整不够，需要重复几千次。**

```
第1次：误差0.8
第10次：误差0.5
第100次：误差0.2
第1000次：误差0.05
第10000次：误差0.001（接近完美）
```

每次调整一点点，累积起来就是大幅改进。

## 避坑指南

1. **学习率不能太大** - 步子太大会跳过最优值
2. **训练数据要准确** - 错误标注会误导网络
3. **不要训练过度** - 训练太久会"背答案"（过拟合）

## 课后思考

1. 如果学习率设为0，网络会学到什么？（答案：什么都学不到，权重不变）
2. 为什么需要多次训练而不是一次到位？
3. 如果训练数据标注错了（叶子标成花朵），网络会学到什么？

# A.3 语言生成：预测下一个字

## 学习目标

- 理解语言模型如何生成文字
- 用数值演示字符预测过程
- 明白"生成"就是不断预测

## 开篇问题

**神经网络能识别图片、预测天气。**

问题是：**如何让神经网络生成文字？**

答案：预测下一个字，不断重复。

## 核心概念

### 语言生成的本质：预测下一个字符

**类比：完形填空**

```
题目："I love y____"
填空：你会填什么？
答案："ou"（因为"I love you"是常见短语）
```

神经网络做同样的事：
- 输入："I love y"
- 输出：每个字母的概率
- 选择概率最大的字母："o"

### 输入层的变化：从图片到文字

**识别图片：输入像素值**
```
图片 → 像素值（0-255） → 数字输入
```

**生成文字：输入字符编号**
```
文字 → 字符编号 → 数字输入

编码方案：
  a=1, b=2, c=3, ..., z=26
  空格=27, 句号=28
```

### 输出层的变化：从2个到26个

**识别叶子/花朵：输出2个概率**
```
叶子概率，花朵概率
```

**生成字母：输出26+个概率**
```
a的概率, b的概率, c的概率, ..., z的概率, 空格的概率
```

## 数值演示：生成一个字母

### 输入编码

```
输入文字："I love y"

编码：
  'I' → 9
  '空格' → 27
  'l' → 12
  'o' → 15
  'v' → 22
  'e' → 5
  '空格' → 27
  'y' → 25

输入向量：[9, 27, 12, 15, 22, 5, 27, 25]
```

### 网络计算

```
输入层：8个神经元（对应8个字符）

隐藏层：计算加权和、激活

输出层：26个神经元（对应26个字母）

输出概率：
  a=0.05, b=0.08, c=0.03, ..., o=0.80, ..., z=0.02
```

### 选择下一个字符

```
概率最大的字母：o（概率0.80）

选择：'o'

新的输入："I love yo"
```

### 重复生成

```
第1轮：输入"I love y" → 输出"o" → 新输入"I love yo"
第2轮：输入"I love yo" → 输出"u" → 新输入"I love you"
第3轮：输入"I love you" → 输出"空格" → 新输入"I love you "
第4轮：输入"I love you " → 输出"too" → 新输入"I love you too"

最终生成："I love you too"
```

## 语言生成的关键

**本质：预测下一个字符，拼接，再预测。**

```
预测 → 拼接 → 预测 → 拼接 → 预测 → ...
```

这就是 ChatGPT 等语言模型的核心原理。

## 避坑指南

1. **不是真正理解语言** - 网络只看字符编号，不理解含义
2. **依赖训练数据** - 如果训练数据里"I love you"多，网络倾向生成这个
3. **可能生成重复** - 没有约束，可能陷入循环

## 课后思考

1. 如果输入"I hate y"，网络会预测什么字母？
2. 为什么网络能生成"I love you"而不是"I love xyz"？
3. 如何让网络生成更长的句子？

# A.4 嵌入：字变成数字向量

## 学习目标

- 理解为什么简单编号不够
- 明白嵌入的本质：相似字向量相近
- 知道嵌入让网络"理解"语义

## 开篇问题

**上一节我们用简单编号：a=1, b=2, ..., z=26。**

问题：**"猫"和"狗"编号是 13 和 14，它们的关系和"猫"与"汽车"一样吗？**

答案：不应该一样。"猫"和"狗"都是动物，应该更相似。

## 核心概念

### 简单编号的问题

**编号：a=1, b=2, ..., 猫=13, 狗=14, 汽车=100**

```
猫的编号：13
狗的编号：14
汽车的编号：100

问题：
  13 和 14 的差 = 1
  13 和 100 的差 = 87
  
差距表示关系吗？
  猫和狗差1 → 它们很相似？
  猫和汽车差87 → 它们很不相似？
  
答案：编号差距不代表语义关系
```

### 嵌入：用向量表示语义

**核心思想：相似的字，向量相近。**

```
简单编号（1个数字）：
  猫=13, 狗=14, 汽车=100
  
嵌入向量（多个数字）：
  猫=[0.2, 0.8, 0.1, 0.3]  （4维向量）
  狗=[0.3, 0.7, 0.2, 0.4]  （和猫接近）
  汽车=[0.9, 0.1, 0.5, 0.2] （和猫、狗都远）
```

### 向量距离表示相似度

```
猫向量：[0.2, 0.8, 0.1, 0.3]
狗向量：[0.3, 0.7, 0.2, 0.4]

计算距离（简化版）：
  差值 = |0.2-0.3| + |0.8-0.7| + |0.1-0.2| + |0.3-0.4|
       = 0.1 + 0.1 + 0.1 + 0.1
       = 0.4（距离小 → 相似）

汽车向量：[0.9, 0.1, 0.5, 0.2]
猫向量：[0.2, 0.8, 0.1, 0.3]

计算距离：
  差值 = |0.9-0.2| + |0.1-0.8| + |0.5-0.1| + |0.2-0.3|
       = 0.7 + 0.7 + 0.4 + 0.1
       = 1.9（距离大 → 不相似）
```

**关键：向量距离反映语义相似度。**

## 为什么需要嵌入？

**简单编号：网络不理解关系**

```
输入"猫" → 编号13 → 网络处理13
输入"狗" → 编号14 → 网络处理14

网络只知道13和14是两个数字
不知道13和14都是动物
```

**嵌入向量：网络能"理解"关系**

```
输入"猫" → 向量[0.2, 0.8, 0.1, 0.3] → 网络处理
输入"狗" → 向量[0.3, 0.7, 0.2, 0.4] → 网络处理

网络看到两个向量很接近 → 知道它们相似
```

## 嵌入的神奇性质

**向量运算：**

```
向量("国王") - 向量("男") + 向量("女") ≈ 向量("女王")

类比推理：
  中国 : 北京 = 日本 : ?
  
向量计算：
  向量("中国") -向量("北京") +向量("日本") ≈ 向量("东京")
```

**这说明：向量捕获了语义关系。**

## 避坑指南

1. **嵌入需要学习** - 不是手动设置，是网络学出来的
2. **维度要适中** - 太小语义不够，太大计算慢（常用100-300维）
3. **多义词问题** - "苹果"（水果）和"苹果"（公司）向量可能混淆

## 课后思考

1. 为什么"出现在相似上下文的字，语义相似"？
2. 嵌入向量能否区分"苹果"（水果）和"苹果"（公司）？
3. 如果用嵌入向量而不是简单编号，语言生成会有什么改进？

## 本章总结

### LLM 的核心原理（中学数学版）

**1. 神经网络：数字的黑盒**
- 输入数字 → 加权和 → 激活 → 输出数字
- 网络不理解含义，只处理数字

**2. 训练：找到好权重**
- 拿数据 → 计算输出 → 算误差 → 调权重 → 重复
- 学习就是不断试错调整

**3. 语言生成：预测下一个字**
- 输入文字 → 输出概率 → 选择字符 → 拼接 → 重复
- 生成就是不断预测

**4. 嵌入：字变成向量**
- 相似字向量相近
- 向量距离反映语义关系

**全部只需要加法、乘法、简单判断。**
